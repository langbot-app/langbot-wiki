---
title: "セキュリティプロファイル"
---

プロファイルはサンドボックスのデフォルト動作とセキュリティ制限を定義します。プリセットされたプロファイルにより、各パラメータを手動で調整することなく、サンドボックスのセキュリティレベルを素早く設定できます。

## 設計思想

プロファイルは2つのコア目的を果たします：

1. **デフォルト値の提供**：エージェントのツール呼び出しパラメータに合理的なデフォルトを設定
2. **重要フィールドのロック**：LLMがプロンプトで権限昇格しようとしても、ツールパラメータで上書きできないようにする

<Tip>
フィールドがロックされている場合、エージェントが`exec`ツール呼び出しで指定したパラメータは無視され、常にプロファイルのデフォルト値が使用されます。これにより、LLMがプロンプトされてもセキュリティ制限を回避できなくなります。
</Tip>

## 組み込みプロファイル

LangBotは4つの組み込みプロファイルを提供：

| プロファイル | ネットワーク | ファイルマウント | リソース | 使用例 |
|-------------|-------------|----------------|----------|--------|
| `default` | オフ | 読み書き | 標準 | 一般的な使用、ほとんどのユーザーに推奨 |
| `offline_readonly` | オフ | 読み取り専用 | 少なめ | 高セキュリティ、ファイル読み取りのみ |
| `network_basic` | オン | 読み書き | 標準 | ネットワークアクセスが必要なスクリプト |
| `network_extended` | オン | 読み書き | 多め | 完全なアクセス権限が必要なネットワークタスク |

## プロファイル詳細

### default

ほとんどのシナリオに適したバランスの取れた設定：ネットワークなし、読み書きマウント、標準リソース制限。

```yaml
box:
  local:
    profile: 'default'
```

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| ネットワーク | オフ | ネットワークアクセスなし |
| ファイルマウント | 読み書き | マウントされたディレクトリの読み書き可能 |
| CPU制限 | 1コア | |
| メモリ制限 | 512 MB | |
| PID制限 | 128 | |
| 読み取り専用ルートFS | はい | コンテナのルートディレクトリは読み取り専用 |
| 最大タイムアウト | 120秒 | |

**使用例**：
- Pythonデータ処理スクリプトの実行
- ファイルの読み書き操作
- ネットワーク不要なタスク

---

### offline_readonly

機密データを扱うための最高セキュリティレベル。重要フィールドがロックされ、LLMがプロンプトで権限昇格できないようにします。

```yaml
box:
  local:
    profile: 'offline_readonly'
```

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| ネットワーク | オフ | ネットワークアクセスなし |
| ファイルマウント | 読み取り専用 | 読み取りのみ、変更不可 |
| CPU制限 | 0.5コア | |
| メモリ制限 | 256 MB | |
| PID制限 | 64 | |
| 読み取り専用ルートFS | はい | |
| 最大タイムアウト | 60秒 | |

**ロックされたフィールド**: `network`、`host_path_mode`、`read_only_rootfs`

LLMがツール呼び出しで`network: 'on'`や`host_path_mode: 'rw'`を渡そうとしても無視され、常にプロファイルの安全な値が使用されます。

**使用例**：
- 機密設定ファイルの読み取り
- ユーザーデータの分析
- ファイル内容を表示するだけでよいシナリオ

<Warning>
このプロファイルでは、エージェントはファイルを作成または変更できません。ファイルへの書き込みが必要なタスクでは`default`または`network_basic`を使用してください。
</Warning>

---

### network_basic

ネットワークアクセスが必要な場合に推奨。

```yaml
box:
  local:
    profile: 'network_basic'
```

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| ネットワーク | オン | ネットワークアクセス許可 |
| ファイルマウント | 読み書き | |
| CPU制限 | 1コア | |
| メモリ制限 | 512 MB | |
| PID制限 | 128 | |
| 読み取り専用ルートFS | はい | |
| 最大タイムアウト | 120秒 | |

**使用例**：
- 外部APIの呼び出し
- ファイルのダウンロード
- 簡単なWebスクレイピング
- Pythonパッケージのインストール

---

### network_extended

完全な権限が必要なネットワークタスク用。

```yaml
box:
  local:
    profile: 'network_extended'
```

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| ネットワーク | オン | 完全なネットワークアクセス |
| ファイルマウント | 読み書き | |
| CPU制限 | 2コア | |
| メモリ制限 | 1024 MB | |
| PID制限 | 256 | |
| 読み取り専用ルートFS | いいえ | ルートディレクトリに書き込み可能 |
| 最大タイムアウト | 300秒 | |

**使用例**：
- 長時間実行されるスクリプト
- システムパッケージのインストールが必要なタスク
- 複雑なネットワーク操作
- より多くのリソースが必要なタスク

<Warning>
このプロファイルはエージェントにより多くの権限を与えます。LLMプロバイダーを信頼していることを確認してください。
</Warning>

## リソース制限の詳細

### CPU制限

サンドボックスが使用できるCPU時間を制御：

| 値 | 説明 |
|-----|------|
| 0.5 | 最大0.5 CPUコア |
| 1.0 | 最大1 CPUコア |
| 2.0 | 最大2 CPUコア |

### メモリ制限

サンドボックスが使用できるメモリを制御：

| 値 | 説明 |
|-----|------|
| 256 MB | 簡単なスクリプト向け |
| 512 MB | デフォルト、ほとんどのタスク向け |
| 1024 MB | データ処理、ML向け |

制限を超えるとプロセスは終了されます。

### PID制限

サンドボックス内で作成できるプロセス数を制御：

| 値 | 説明 |
|-----|------|
| 64 | 最小、fork爆弾を防止 |
| 128 | デフォルト |
| 256 | マルチプロセスタスク向け |

### タイムアウト制限

単一コマンドの最大実行時間：

- エージェントは`timeout_sec`を指定可能だが、プロファイルの`max_timeout_sec`を超えることはできない
- タイムアウト後、コマンドは強制終了される

## ロックされたフィールド

プロファイルは特定のフィールドをロックして、エージェントがツールパラメータで上書きできないようにします：

```python
# プロファイル定義のlockedフィールド
locked=frozenset({'network', 'host_path_mode', 'read_only_rootfs'})
```

フィールドがロックされている場合：
- エージェントが`exec`ツール呼び出しで指定したパラメータは無視される
- 常にプロファイルのデフォルト値が使用される

これにより、LLMがプロンプトされてもセキュリティ制限を回避できなくなります。

---

## カスタム設定

プロファイルはデフォルト値を提供します。`config.yaml`で上書き可能：

```yaml
box:
  local:
    profile: 'default'
    # デフォルトを上書き
    workspace_quota_mb: 100    # ワークスペースサイズを100MBに制限
```

### ワークスペースクォータ

サンドボックスワークスペースが使用するディスク容量を制限：

```yaml
box:
  local:
    workspace_quota_mb: 500  # 500 MB
```

クォータを超えた場合：
- 新しいセッションの作成が拒否される
- 既存のセッションが破棄される可能性がある

## セキュリティベストプラクティス

### 1. 最小権限の原則

実際のニーズに基づいて、最も低い権限のプロファイルを選択：

```
offline_readonly → default → network_basic → network_extended
   (最も安全)                                    (最も緩やか)
```

### 2. マウントディレクトリの制限

`allowed_mount_roots`を設定して、エージェントがアクセスできるホストディレクトリを制限：

```yaml
box:
  local:
    allowed_mount_roots:
      - './data/box'
      - '/tmp'
```

エージェントはこれらのディレクトリまたはそのサブディレクトリのみマウント可能。

### 3. ディスククォータの使用

エージェントが過度なディスク容量を消費しないようにする：

```yaml
box:
  local:
    workspace_quota_mb: 100
```

### 4. サンドボックスステータスの監視

APIでサンドボックスのステータスとエラーを確認：

```
GET /api/v1/box/status
GET /api/v1/box/errors
```
