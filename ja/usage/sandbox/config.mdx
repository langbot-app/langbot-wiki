title: "設定リファレンス"
---

サンドボックス設定オプションの完全版。

## 設定場所

サンドボックス設定は`data/config.yaml`の`box`セクションにあります：

```yaml
box:
  enabled: true
  backend: 'local'
  runtime:
    endpoint: ''
  local:
    profile: 'default'
    image: ''
    host_root: './data/box'
    default_workspace: ''
    skills_root: 'skills'
    allowed_mount_roots:
      - './data/box'
      - '/tmp'
    workspace_quota_mb: null
  e2b:
    api_key: ''
    api_url: ''
    template: ''
```

---

## トップレベル設定

### `enabled`

サンドボックスランタイムのマスタースイッチ。

| 値 | 説明 |
|-----|------|
| `true` | サンドボックスを有効化（デフォルト） |
| `false` | サンドボックスを完全に無効化 |

デフォルト：`true`

`false` に設定すると、LangBot はリモート Box Runtime に接続せず、ローカルの stdio Box サブプロセスも起動しません。サンドボックスに依存するすべての機能が無効になります：内蔵ツール（`exec`/`read`/`write`/`edit`/`glob`/`grep`）、`activate` スキルツール、スキルの新規作成・編集、および stdio モードの MCP Server。Skills は読み取り専用での一覧表示は可能で、http/sse モードの MCP Server は影響を受けません。

### `backend`

サンドボックスバックエンドタイプを強制指定。

| 値 | 説明 |
|-------|-------------|
| 空/未設定 | すべてのバックエンドを試行（Docker → Nsjail → E2B）、最初に利用可能なものを使用 |
| `local` | ローカルバックエンドのみ試行（Docker → Nsjail）、E2Bは試行しない |
| `docker` | Dockerのみ強制 |
| `nsjail` | Nsjailのみ強制 |
| `e2b` | E2Bクラウドサンドボックスのみ強制 |

デフォルト：`local`

<Note>
これは**強制値**であり、優先度リストではありません。例えば`local`に設定した場合、E2Bが利用可能でも使用されません。
</Note>

### `runtime.endpoint`

外部Box RuntimeのWebSocket URL。

| 値 | 説明 |
|-------|-------------|
| 空 | 自動検出（ローカルではstdio、コンテナではWebSocket） |
| `ws://host:port` | 指定されたRuntimeに接続 |

デフォルト：空

<Note>
ほとんどの場合、これを設定する必要はありません。Box Runtimeを独立してデプロイする場合にのみ使用します。
</Note>

## local設定

### `local.profile`

セキュリティプロファイル名。

| 値 | 説明 |
|-------|-------------|
| `default` | デフォルトプロファイル、ネットワークなし、読み書きマウント |
| `offline_readonly` | 高セキュリティ、ネットワークなし、読み取り専用マウント |
| `network_basic` | ネットワーク有効、読み書きマウント |
| `network_extended` | 完全なネットワーク、読み書きマウント、より多くのリソース |

デフォルト：`default`

詳細は[セキュリティプロファイル](./profiles)を参照。

### `local.image`

カスタムサンドボックスイメージ。

| 値 | 説明 |
|-------|-------------|
| 空 | デフォルトイメージ`rockchin/langbot-sandbox:latest`を使用 |
| `python:3.11-slim` | Python 3.11イメージを使用 |
| `node:20-slim` | Node.js 20イメージを使用 |

デフォルト：空

<Note>
Dockerバックエンドのみ適用されます。Nsjailはホストシステム環境を使用するため、カスタムイメージはサポートされません。
</Note>

### `local.host_root`

ホスト上のサンドボックスワーキングディレクトリのルートパス。

サンドボックスワークスペースのファイルはこのディレクトリ下に保存されます。

| 値 | 説明 |
|-------|-------------|
| `./data/box` | LangBot プロセスの作業ディレクトリからの相対パス |
| `/var/lib/langbot/box` | 絶対パス |

デフォルト：`./data/box`

### `local.default_workspace`

デフォルトワークスペース名。

| 値 | 説明 |
|-------|-------------|
| 空 | `<host_root>/default`を使用 |
| `my-workspace` | `<host_root>/my-workspace`を使用 |

デフォルト：空

### `local.skills_root`

Box Runtime が管理する Skill パッケージディレクトリ。

| 値 | 説明 |
|-------|-------------|
| `skills` | `<host_root>/skills` を使用 |
| `/var/lib/langbot/skills` | 絶対パスを使用 |

デフォルト：`skills`

<Note>
Skills は完全に Box Runtime が管理し、Box がスキルの唯一の供給元です。Box が利用できない場合（設定で無効化されている、または接続に失敗した場合）、スキル一覧は空になります——`data/skills` から読み込むローカルフォールバックは存在しません。
</Note>

### `local.allowed_mount_roots`

エージェントがマウントできるホストディレクトリのホワイトリスト。

エージェントがリクエストした`host_path`はこのリスト内のいずれかのディレクトリ下にある必要があります。

```yaml
allowed_mount_roots:
  - './data/box'     # サンドボックスワーキングディレクトリのマウントを許可
  - '/tmp'           # 一時ディレクトリのマウントを許可
  - '/home/user/projects'  # プロジェクトディレクトリのマウントを許可
```

デフォルト：`['<host_root>']`（サンドボックスワーキングディレクトリのみマウント許可）

### `local.workspace_quota_mb`

ワークスペースのディスククォータ（MB単位）。

| 値 | 説明 |
|-------|-------------|
| `null` | プロファイルのデフォルトを使用（通常は0、無制限） |
| `0` | 制限なし |
| `100` | 100 MBに制限 |
| `500` | 500 MBに制限 |

デフォルト：`null`

<Note>
このクォータは強制されます。超過した場合、既存のセッションが破棄される可能性があります。
</Note>

## e2b設定

### `e2b.api_key`

E2B API Key。

`E2B_API_KEY`環境変数でも設定可能で、こちらが優先されます。

| ソース | 優先度 |
|--------|----------|
| `E2B_API_KEY`環境変数 | 高 |
| `config.yaml`の`e2b.api_key` | 低 |

### `e2b.api_url`

カスタムE2B API URL。

自己ホストE2Bサービスに接続する場合に使用。`E2B_API_URL`環境変数でも設定可能。

| 値 | 説明 |
|-------|-------------|
| 空 | 公式E2Bサービスを使用 |
| `https://your-e2b.example.com` | 自己ホストサービスのアドレス |

### `e2b.template`

デフォルトE2BテンプレートID。

| 値 | 説明 |
|-------|-------------|
| 空 | E2Bデフォルトテンプレートを使用 |
| `base` | ベーステンプレート |
| `python-3.11` | Python 3.11テンプレート |
| カスタムID | E2Bで作成したカスタムテンプレート |

## 環境変数

LangBot は統一された環境変数の上書き規約を採用しています：`SECTION__SUBSECTION__KEY`（ダブルアンダースコア区切り）が `config.yaml` の対応するフィールドを上書きします。`box` 設定項目は次のように設定できます：

| 環境変数 | 対応する設定項目 |
|---------------------|----------------------|
| `BOX__BACKEND` | `box.backend` |
| `BOX__LOCAL__PROFILE` | `box.local.profile` |
| `BOX__LOCAL__IMAGE` | `box.local.image` |
| `BOX__LOCAL__HOST_ROOT` | `box.local.host_root` |
| `BOX__LOCAL__DEFAULT_WORKSPACE` | `box.local.default_workspace` |
| `BOX__LOCAL__SKILLS_ROOT` | `box.local.skills_root` |
| `BOX__LOCAL__ALLOWED_MOUNT_ROOTS` | `box.local.allowed_mount_roots`。カンマ区切り |
| `BOX__LOCAL__WORKSPACE_QUOTA_MB` | `box.local.workspace_quota_mb` |
| `BOX__E2B__API_KEY` | `box.e2b.api_key` |
| `BOX__E2B__API_URL` | `box.e2b.api_url` |
| `BOX__E2B__TEMPLATE` | `box.e2b.template` |
| `BOX_BACKEND` | Box Runtime バックエンドを強制し、`box.backend` より優先 |

<Note>
E2B SDKは`E2B_API_KEY`と`E2B_API_URL`環境変数もサポートしており、`config.yaml`より優先されます。

Box Runtime プロセス（コンテナ化デプロイ時の `langbot_box` サービス）は `config.yaml` や環境変数を**読み込みません**。その設定は LangBot メインプロセスが INIT RPC を通じて下げ渡します。したがって `BOX__*` などの環境変数は `langbot` サービスにのみ設定すれば十分です。
</Note>

## Docker Compose例

Docker Compose では、Box Runtime を独立した `langbot_box` サービスとして実行します。これは**オプションサービス**であり、compose profile で制御されます——`docker compose --profile box up`（または `--profile all`）で起動する必要があります。`langbot_box` は `langbot` と同じホスト上の Box ルートを共有します：

```yaml
# docker-compose.yaml
services:
  langbot_box:
    image: rockchin/langbot:latest
    # オプションサービス：compose profile で起動の有無を制御
    profiles: ["box", "all"]
    volumes:
      # langbot_box はホスト Docker socket 経由でサンドボックスコンテナを作成するため、
      # ソースパスとターゲットパスを同一にします。
      - ${LANGBOT_BOX_ROOT:-${PWD}/data/box}:${LANGBOT_BOX_ROOT:-${PWD}/data/box}
      - /var/run/docker.sock:/var/run/docker.sock
    # langbot_box は config.yaml や環境変数を読み込みません。設定は langbot が INIT RPC で下げ渡します。
    # ここに BOX__* / LANGBOT_BOX_* 変数を追加しないでください——無視されます。
    command: ["uv", "run", "--no-sync", "-m", "langbot_plugin.cli.__init__", "box"]

  langbot:
    image: rockchin/langbot:latest
    volumes:
      - ./data:/app/data
    environment:
      # 統一された環境変数の上書き規約。これらの box.local.* 設定は langbot が
      # INIT RPC を通じて Box Runtime へ転送します。
      - BOX__LOCAL__HOST_ROOT=${LANGBOT_BOX_ROOT:-${PWD}/data/box}
      - BOX__LOCAL__DEFAULT_WORKSPACE=default
      - BOX__LOCAL__SKILLS_ROOT=skills
      - BOX__LOCAL__ALLOWED_MOUNT_ROOTS=${LANGBOT_BOX_ROOT:-${PWD}/data/box}
```

<Warning>
`langbot_box` は Docker daemon へのアクセスが必要です。信頼できる環境でのみ `docker.sock` をマウントしてください。`langbot_box` はホストの Docker socket でサンドボックスコンテナを作成するため、Box ルートのホストパスとコンテナ内パスは同一にする必要があります。
</Warning>

## 完全な設定例

### 基本設定

```yaml
box:
  backend: 'local'
  local:
    profile: 'default'
    host_root: './data/box'
```

### 開発環境

```yaml
box:
  backend: 'local'
  local:
    profile: 'network_extended'
    host_root: './data/box'
    allowed_mount_roots:
      - './data/box'
      - '/tmp'
      - './projects'
```

### 高セキュリティ設定

```yaml
box:
  backend: 'local'
  local:
    profile: 'offline_readonly'
    host_root: './data/box'
    allowed_mount_roots:
      - './data/box'
    workspace_quota_mb: 50
```

### E2Bクラウドサンドボックス

```yaml
box:
  backend: 'e2b'
  e2b:
    api_key: 'e2b_xxxxx'
    template: 'python-3.11'
```
