---
title: "サンドボックス設定"
---

サンドボックス設定は LangBot メインプロセスが管理し、Box Runtime が実際の実行を担当します。通常のデプロイでは、バックエンド、ワークスペースディレクトリ、セキュリティプロファイルだけを決めれば十分です。

## 推奨設定

```yaml
box:
  enabled: true
  backend: 'local'
  local:
    profile: 'default'
    host_root: './data/box'
    skills_root: 'skills'
```

- `enabled`: 全体スイッチ。`false` にすると、サンドボックスツール、Skill の追加/編集、stdio MCP ホスティングなど Box 依存機能は利用できません。
- `backend`: バックエンド選択。デフォルトの `local` はローカルの Docker/Nsjail のみを試します。
- `local.profile`: ネットワーク、マウント、リソース制限を制御するセキュリティプロファイル。
- `local.host_root`: ホスト上のサンドボックス作業ディレクトリ。サンドボックス内では `/workspace` にマッピングされます。
- `local.skills_root`: Skill パッケージディレクトリ。相対パスは `host_root` 配下に解決され、デフォルトは `host_root/skills` です。

<Note>
現在の実装では、Skills は Box Runtime が管理する skill store からのみ読み込まれます。Box Runtime またはバックエンドが利用できない場合、Skill 一覧は空になり、作成、インポート、編集、`register_skill` は利用できません。`data/skills/` の自動スキャンにはフォールバックしません。
</Note>

## バックエンド選択

| バックエンド | 用途 | 説明 |
|--------------|------|------|
| `local` | 推奨デフォルト | Docker、Nsjail の順に試す。E2B は試さない |
| `docker` | 本番環境、コンテナ分離 | Docker daemon が必要。カスタムイメージ対応 |
| `nsjail` | 軽量 Linux 分離 | `nsjail` が必要。カスタムイメージ非対応 |
| `e2b` | クラウドサンドボックス | E2B API Key が必要。ローカルコンテナ環境は不要 |

`backend` は強制値であり、優先順位リストではありません。たとえば `local` にすると、E2B が設定されていても使用されません。

## セキュリティプロファイル

| Profile | ネットワーク | マウント | リソース | 推奨場面 |
|---------|--------------|----------|----------|----------|
| `default` | 無効 | 読み書き | デフォルト制限 | 通常のデフォルト |
| `offline_readonly` | 無効 | 読み取り専用 | より厳格 | 信頼できないファイルの読み取り |
| `network_basic` | 基本ネットワーク | 読み書き | デフォルト制限 | API アクセスや依存関係のダウンロード |
| `network_extended` | 完全ネットワーク | 読み書き | より緩い | 開発、デバッグ、複雑なタスク |

最小権限を優先してください。ネットワークが不要なら `default` または `offline_readonly` を使い、`allowed_mount_roots` には必要なディレクトリだけを追加します。

## よく使う設定項目

| 設定項目 | デフォルト | 説明 |
|----------|------------|------|
| `runtime.endpoint` | 空 | 外部 Box Runtime の WebSocket URL。通常は不要 |
| `local.image` | 空 | Docker バックエンドのカスタムイメージ。空ならデフォルトイメージ |
| `local.default_workspace` | 空 | デフォルトワークスペース名。空なら `host_root/default` |
| `local.allowed_mount_roots` | `host_root` | Agent がマウントを要求できるホストディレクトリ |
| `local.workspace_quota_mb` | `null` | ワークスペースのディスククォータ。`null` は Profile デフォルト |
| `e2b.api_key` | 空 | E2B API Key。`E2B_API_KEY` 環境変数でも設定可能 |
| `e2b.api_url` | 空 | セルフホスト E2B のエンドポイント。`E2B_API_URL` でも設定可能 |
| `e2b.template` | 空 | デフォルト E2B テンプレート ID |

## Docker Compose

Docker Compose では、Box 設定を `langbot` サービスに設定します。LangBot は有効な設定を INIT RPC 経由で `langbot_box` に渡します。

```yaml
services:
  langbot_box:
    image: rockchin/langbot:latest
    container_name: langbot_box
    profiles: ["box", "all"]
    volumes:
      - ${LANGBOT_BOX_ROOT:-${PWD}/data/box}:${LANGBOT_BOX_ROOT:-${PWD}/data/box}
      - /var/run/docker.sock:/var/run/docker.sock
    command: ["uv", "run", "--no-sync", "-m", "langbot_plugin.cli.__init__", "box"]

  langbot:
    image: rockchin/langbot:latest
    volumes:
      - ./data:/app/data
    environment:
      - BOX__LOCAL__HOST_ROOT=${LANGBOT_BOX_ROOT:-${PWD}/data/box}
      - BOX__LOCAL__SKILLS_ROOT=skills
      - BOX__LOCAL__ALLOWED_MOUNT_ROOTS=${LANGBOT_BOX_ROOT:-${PWD}/data/box}
```

<Warning>
`langbot_box` は Docker daemon へアクセスする必要があります。`docker.sock` は信頼できる環境でのみマウントし、Box ルートパスはホスト側と `langbot_box` コンテナ内で同一にしてください。
</Warning>

## 環境変数

| 環境変数 | 書き込み先 |
|----------|------------|
| `BOX__ENABLED` | `box.enabled` |
| `BOX__BACKEND` | `box.backend` |
| `BOX__LOCAL__PROFILE` | `box.local.profile` |
| `BOX__LOCAL__IMAGE` | `box.local.image` |
| `BOX__LOCAL__HOST_ROOT` | `box.local.host_root` |
| `BOX__LOCAL__DEFAULT_WORKSPACE` | `box.local.default_workspace` |
| `BOX__LOCAL__SKILLS_ROOT` | `box.local.skills_root` |
| `BOX__LOCAL__ALLOWED_MOUNT_ROOTS` | `box.local.allowed_mount_roots`。カンマ区切り |
| `BOX__LOCAL__WORKSPACE_QUOTA_MB` | `box.local.workspace_quota_mb` |
| `BOX__E2B__API_KEY` | `box.e2b.api_key` |
| `BOX__E2B__API_URL` | `box.e2b.api_url` |
| `BOX__E2B__TEMPLATE` | `box.e2b.template` |

<Note>
`langbot_box` サービスに `LANGBOT_BOX_*` や `BOX__*` を設定しないでください。Box Runtime はこれらの変数を直接読み取りません。
</Note>
