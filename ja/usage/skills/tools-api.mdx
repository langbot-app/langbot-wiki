---
title: "スキルツール API"
---

Skills システムは Agent がスキルを起動・登録するための2つのツールを提供します。Claude Code のプログレッシブディスクロージャーパターンに従い、Tool Call メカニズムでスキルを起動します。

## ツール概要

| ツール | 機能 | サンドボックス必要 |
|--------|------|-------------------|
| `activate` | スキルを起動 | 必要 |
| `register_skill` | サンドボックスディレクトリからスキルを登録 | 必要 |

<Note>
これらのツールはサンドボックスバックエンド（Docker/Nsjail/E2B）が利用可能である必要があります。バックエンドがない場合、ツールは Agent に提供されません。`activate` から見えるのは、現在のパイプラインで許可された Skills だけです。
</Note>

---

## activate

スキルを起動し、スキルの命令を現在の会話コンテキストに注入します。

### 使用場面

- 既存のスキルを起動して専門能力を取得
- スキルの完全な命令とランタイムコンテキストを取得

### パラメータ

| パラメータ | 型 | 必須 | 説明 |
|------------|-----|------|------|
| `skill_name` | string | はい | 起動するスキル名（引数不要） |

### 仕組み

1. Agent が `activate` ツールをスキル名で呼び出し（スキルは現在のパイプラインから見える必要があります）
2. システムがスキルの SKILL.md 内容を Tool Result として返す
3. スキルパッケージが `/workspace/.skills/{skill-name}/` にマウントされる
4. Agent は `exec`/`read`/`write`/`edit`/`glob`/`grep` でスキルファイルを操作可能

### 例

```json
{
  "skill_name": "pdf"
}
```

### 戻り値

```json
{
  "activated": true,
  "skill_name": "pdf",
  "mount_path": "/workspace/.skills/pdf",
  "content": "<command-message>The \"pdf\" skill is activated</command-message>\n<skill-activation>\n<skill-name>pdf</skill-name>\n<mount-path>/workspace/.skills/pdf</mount-path>\n<package-root>/path/to/skill</package-root>\n\n## Instructions\n...\n\n## Runtime Context\nThe skill package is mounted at /workspace/.skills/pdf. Use the standard tools to interact with it:\n- Use `read` to inspect files under /workspace/.skills/pdf\n- Use `exec` with workdir set to /workspace/.skills/pdf to run commands in that package\n- Use `write` and `edit` on that path when the instructions require updating files\n</skill-activation>"
}
```

### エラー

- `skill_name is required`: スキル名が提供されていない
- `Skill "xxx" not found`: スキルが存在しない

## register_skill

サンドボックスワークスペースのディレクトリからスキルパッケージを LangBot スキルストアに登録します。サンドボックスバックエンドが利用可能な場合、登録先は Box Runtime が管理し、デフォルトは `<box.local.host_root>/skills` です。

### 使用場面

- サンドボックスで新しいスキルを作成後に登録
- 事前に準備したスキルパッケージをインポート

### 前提条件

- サンドボックス機能が有効
- ターゲットディレクトリが `/workspace` 下に存在
- ディレクトリ、または 2 階層以内のサブディレクトリに `SKILL.md` が含まれている

### パラメータ

| パラメータ | 型 | 必須 | 説明 |
|------------|-----|------|------|
| `path` | string | はい | サンドボックス内のディレクトリパス（`/workspace` 下。このディレクトリまたは 2 階層以内のサブディレクトリに SKILL.md が必要） |
| `name` | string | いいえ | スキル名を上書き、デフォルトは SKILL.md の名前またはディレクトリ名 |
| `display_name` | string | いいえ | 表示名を上書き |
| `description` | string | いいえ | 説明を上書き |
| `instructions` | string | いいえ | 命令内容を上書き |

### ワークフロー

```
1. サンドボックスでスキルディレクトリを準備
   /workspace/my-skill/
   ├── SKILL.md
   └── scripts/
       └── main.py

2. register_skill を呼び出し

3. スキルがスキルストアに登録される
   - activate ツールで起動可能
   - デフォルトでは Box Runtime 管理の skills root に書き込まれる
```

### 例

```json
{
  "path": "/workspace/my-skill",
  "name": "data-processor",
  "description": "データファイルを処理・変換"
}
```

### 戻り値

```json
{
  "registered": true,
  "skill_name": "data-processor",
  "source_path": "/workspace/my-skill",
  "skill": {
    "name": "data-processor",
    "display_name": "データ処理ツール",
    "description": "データファイルを処理・変換",
    "instructions": "...",
    "package_root": "/path/to/box/skills/data-processor"
  }
}
```

### エラー

- `path is required`: パスが提供されていない
- `path must be under /workspace`: パスがワークスペース内にない
- `Directory does not exist`: ディレクトリが存在しない
- `No SKILL.md found in ...`: ディレクトリまたは 2 階層以内のサブディレクトリに SKILL.md がない

## 起動メカニズム

Skills システムはテキストマーカーではなく Tool Call メカニズムでスキルを起動します。これは Claude Code の設計に沿っています。

### 起動方法

Agent は `activate` ツールを呼び出してスキルを起動：

```json
{
  "skill_name": "pdf"
}
```

### 起動後の効果

1. **命令注入**: SKILL.md 内容が Tool Result として返され、コンテキストに注入
2. **パスマウント**: スキルパッケージが `/workspace/.skills/{skill-name}/` にマウント
3. **ファイル操作**: Agent は標準ツールでスキルファイルを操作可能
   - `read`: スキルファイルを読み取り
   - `exec`: スキルディレクトリでコマンドを実行
   - `write`/`edit`: スキルファイルを修正

### 設計の利点

| 特徴 | 説明 |
|------|------|
| **KV Cache 保護** | スキル命令は Tool Result 経由でオンデマンドでロードされ、会話履歴のキャッシュ効率に影響しない |
| **プログレッシブディスクロージャー** | ツール説明にはスキルメタデータ（名前、説明）のみ表示、完全な命令は起動後に取得 |
| **ランタイム分離** | 各起動されたスキルは独立したマウントパスを持ち、ファイル競合を回避 |

### テキストマーカーとの比較

| 側面 | 旧方式（テキストマーカー） | 新方式（Tool Call） |
|------|--------------------------|-------------------|
| 起動方法 | `[ACTIVATE_SKILL: name]` | `activate` ツール呼び出し |
| 命令注入 | System Prompt に事前注入 | Tool Result で動的ロード |
| KV Cache | 影響あり | 保護 |
| マルチスキル対応 | 複数マーカー | 複数回呼び出し |

## 新しいスキルの作成

新しいスキルを作成する一般的な方法は 2 つあります：

1. WebUI の Skills ページで作成、編集、GitHub からのインポート、zip パッケージのアップロードを行う
2. Agent に `/workspace` 下でスキルディレクトリを準備させ、`register_skill` を呼び出す

登録後、新しいスキルは `activate` で使用できます。

<Note>
新しいスキルは Box Runtime が管理する skills root に登録されます。Box がスキルの唯一の供給元です——Box が利用できない場合はスキルを新規追加できず、`data/skills/` のローカルフォールバックも存在しません。
</Note>
