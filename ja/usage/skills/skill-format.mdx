---
title: "SKILL.md フォーマット"
---

各 Skill は 1 つのディレクトリで、そのルートに `SKILL.md` を配置します。`SKILL.md` は **メタデータ frontmatter** と **指示本文** の 2 つで構成されます。

## 全体構造

```markdown
---
name: my-skill
display_name: 私のスキル
description: Agent がいつ起動するかを判断するための簡潔な説明
---

## 実行手順

1. ...
2. ...
```

## Frontmatter フィールド

| フィールド | 必須 | 説明 |
|------|------|------|
| `name` | 作成／登録時 | 一意の識別子。1〜64 文字、英数字・ハイフン・アンダースコア。既存ディレクトリを直接スキャンする場合は省略可能で、ディレクトリ名が使用される |
| `description` | 強く推奨 | 簡潔な説明。Agent がこの Skill を見つけられるかを左右する |
| `display_name` | 任意 | 表示名（UI 用） |

その他のよくあるフィールド（`license` / `compatibility` / `metadata` / `allowed-tools`）は現在読み取られませんが、将来の互換性や人間が読むために残しておけます。

### `description` を上手く書く

`description` は Agent が「Skill を選ぶ」ための唯一の判断材料で、`<available_skills>` 一覧に表示されます。説明が曖昧だと、この Skill は起動されないも同然です。

```yaml
# ❌ あいまいすぎる
description: PDF の処理を手伝う

# ✅ 機能、シナリオ、トリガーキーワードを含む
description: PDF からテキストやテーブルを抽出、PDF フォームへの入力、複数 PDF の結合。ユーザーが PDF、フォーム、ドキュメント抽出に言及したときに起動
```

執筆のポイント:
- **機能**（何ができるか）を明確に
- **適用シーン**（どんなときに使うか）を明確に
- **トリガーキーワード**（ユーザーが言いそうな表現）を含める

## 指示本文

frontmatter の後ろは Markdown 本文です。Agent が `activate` を呼び出すと、本文全体が Tool Result として返され、コンテキストに注入されます。

推奨構造（必須ではありません）:

```markdown
## 使用場面
どのようなリクエストにこの Skill が適しているか。

## 実行手順
1. ...
2. ...

## 利用可能なリソース
scripts/、templates/ 配下で利用できるファイルとその用途。

## 出力フォーマット
出力に特定のフォーマットがある場合はここに記載。
```

本文の長さに注意: 500 行以内が目安です。大量のリファレンスはサブディレクトリに置き、必要に応じて `read` ツールで読み込みます。

## 完全な例

```markdown
---
name: api-tester
display_name: API テストツール
description: REST API エンドポイントのテスト、レスポンス検証、パフォーマンスベンチマーク。ユーザーが API テスト、インターフェース検証、負荷テストに言及したときに起動
---

## 使用場面

HTTP リクエストを発行する、レスポンスを検証する、または単一のエンドポイントに対して負荷テストを行う必要があるとき。

## 実行手順

1. ユーザー入力からメソッド、URL、ヘッダー、ボディを抽出
2. `exec` で `scripts/request.py` を呼び出す
3. ステータスコードとレスポンスボディを検証
4. パフォーマンステストが必要な場合は `scripts/benchmark.py` を呼び出す
5. テンプレートに従って結果を出力

## 利用可能なリソース

- `scripts/request.py` — 単発の HTTP リクエスト
- `scripts/benchmark.py` — 並列負荷テスト
- `templates/test_report.md` — レポートテンプレート

## 出力フォーマット

単発: URL、ステータスコード、レスポンス時間、レスポンスボディ要約
バッチ: 成功／失敗件数、平均レスポンス時間、エラー一覧
```

## ディレクトリ構成

最小:

```
my-skill/
└── SKILL.md
```

リソース付き:

```
my-skill/
├── SKILL.md           # 必須
├── scripts/           # 任意: 実行可能スクリプト
├── templates/         # 任意: テンプレート
└── requirements.txt   # 任意: 依存関係（Agent が必要に応じて pip install）
```

起動後、Agent は `/workspace/.skills/{skill-name}/` 経由でこれらのファイルにアクセスします。

## 配置場所

サンドボックスバックエンドが利用可能な場合、Skill は Box Runtime が管理する skill store に保存されます。デフォルトのパス:

```
<box.local.host_root>/skills/<skill-name>/SKILL.md
```

`box.local.skills_root` はデフォルトで相対パス `skills` で、`host_root` 配下に解決されます。Box が利用できない場合、Skill 一覧は空になり、他のディレクトリへのフォールバックはありません。
