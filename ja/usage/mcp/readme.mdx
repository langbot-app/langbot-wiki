LangBot は MCP サーバーのツールを利用できます。MCP サーバーを追加し、Tool Call をサポートするモデルからそれらのツールを呼び出せます。

<Info title="MCP サービスは HTTP Streamable、SSE、stdio モードをサポートしています。">
- MCPサーバーの推奨取得チャネル：
  - [ModelScope](https://modelscope.cn/mcp)
  - https://mcp.so
  - https://composio.dev/toolkits
</Info>

## 実行モード

| モード | 向いている用途 | Runtime との関係 |
|--------|----------------|------------------|
| Streamable HTTP (`mode: "http"`) | Streamable HTTP を使うリモート MCP サービス | LangBot がリモート URL に直接接続する |
| SSE | SSE を使うリモート MCP サービス | LangBot がリモート URL に直接接続する |
| stdio | ローカルコマンドで起動する MCP Server | Box Runtime が利用可能な場合のみ利用可能 |

<Note>
MCP ツールを使うには、パイプラインで `Local Agent` を選び、function calling をサポートするモデルを使う必要があります。Dify、n8n、Langflow、Coze などの外部 Runner は、それぞれのプラットフォームのツール機構を使います。
</Note>

## stdio MCP と Box Runtime

`stdio` 形式の MCP は、Box Runtime が利用可能な場合のみ使用できます。利用可能な場合、LangBot は MCP プロセスを Box 内で起動して接続します。

1. 共有 MCP Box session を作成または再利用する
2. command または args からローカルプロジェクトパスを推測できる場合、共有ワークスペースへ staging する
3. プロジェクトに `requirements.txt`、`pyproject.toml`、`setup.py`、`setup.cfg` があれば、先に依存関係をインストールする
4. WebSocket relay 経由で MCP プロセスの stdin/stdout に接続する
5. MCP サーバーの初期化が成功した後、ツールを Agent の利用可能ツールに追加する

高度な stdio MCP 設定では、`box` override を含められます。

```json
{
  "mode": "stdio",
  "extra_args": {
    "command": "python",
    "args": ["server.py"],
    "env": {},
    "box": {
      "host_path": "/path/to/mcp-project",
      "network": "on",
      "host_path_mode": "ro",
      "startup_timeout_sec": 120,
      "image": "python:3.11-slim"
    }
  }
}
```

主なフィールド:

| フィールド | 説明 |
|------------|------|
| `host_path` | MCP サーバーのホスト側プロジェクトディレクトリ。`box.local.allowed_mount_roots` の範囲内である必要があります |
| `network` | ネットワークを許可するかどうか。依存関係のインストールでよく必要になります |
| `host_path_mode` | マウントモード。デフォルトは `ro` |
| `startup_timeout_sec` | 起動と依存関係インストールの待機時間。デフォルトは 120 秒 |
| `image` | Docker backend で使う実行イメージ |

<Warning>
stdio MCP を Box 内で実行する場合、依存関係インストールとプロセス起動は sandbox profile、ネットワーク権限、リソース制限、マウントホワイトリストの影響を受けます。
</Warning>

## パイプラインで有効化

MCP Server を作成して接続した後、パイプライン拡張設定で許可する必要があります。

- すべての MCP Server を有効化する
- または指定した MCP Server だけをバインドする

指定リストを使う場合、バインドされていない MCP ツールはそのパイプラインの Agent に公開されません。

## よくある問題

| 症状 | 確認すること |
|------|--------------|
| MCP 接続に失敗する | URL、command、args、環境変数 |
| stdio MCP が接続中のままになる | Box Runtime が利用可能か、`startup_timeout_sec` が短すぎないか |
| 依存関係インストールに失敗する | sandbox でネットワークが許可されているか、依存関係ファイルが正しいか |
| ローカルファイルにアクセスできない | `host_path` が存在するか、`allowed_mount_roots` の範囲内か |
| Agent に MCP ツールが見えない | Local Agent、function calling、MCP 接続状態、パイプラインバインド |
