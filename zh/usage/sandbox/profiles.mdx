---
title: "安全预设 (Profiles)"
---

Profile（安全预设）定义了沙箱的默认行为和安全限制。通过预设的 Profile，您可以快速配置沙箱的安全级别，而无需手动调整每个参数。

## 设计思路

Profile 的核心作用是：

1. **提供默认值**：为 Agent 调用工具时的参数设置合理的默认值
2. **锁定关键字段**：防止 LLM 被诱导后通过工具参数提升权限

<Tip>
当字段被锁定时，Agent 在调用 `exec` 工具时指定的参数会被忽略，始终使用 Profile 的默认值。这确保了即使 LLM 被诱导，也无法突破安全限制。
</Tip>

## 内置 Profile

LangBot 提供四种内置 Profile：

| Profile | 网络 | 文件挂载 | 资源 | 适用场景 |
|---------|------|---------|------|---------|
| `default` | 关闭 | 读写 | 标准 | 通用场景，推荐大多数用户 |
| `offline_readonly` | 关闭 | 只读 | 较少 | 高安全要求，只允许读取文件 |
| `network_basic` | 开启 | 读写 | 标准 | 需要网络访问的脚本 |
| `network_extended` | 开启 | 读写 | 更多 | 需要完全访问权限的网络任务 |

---

## Profile 详解

### default（默认）

适用于大多数场景的平衡配置：无网络访问、读写挂载、标准资源限制。

```yaml
box:
  local:
    profile: 'default'
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 网络 | 关闭 | 无法访问网络 |
| 文件挂载 | 读写 | 可以读写挂载的目录 |
| CPU 限制 | 1 核 | |
| 内存限制 | 512 MB | |
| PID 限制 | 128 | |
| 只读根文件系统 | 是 | 容器根目录只读 |
| 最大超时 | 120 秒 | |

**适用场景**：
- 执行 Python 数据处理脚本
- 文件读写操作
- 不需要网络的任务

---

### offline_readonly（离线只读）

最高安全级别，适合处理敏感数据。关键字段被锁定，防止 LLM 被诱导提升权限。

```yaml
box:
  local:
    profile: 'offline_readonly'
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 网络 | 关闭 | 无法访问网络 |
| 文件挂载 | 只读 | 只能读取，无法修改 |
| CPU 限制 | 0.5 核 | |
| 内存限制 | 256 MB | |
| PID 限制 | 64 | |
| 只读根文件系统 | 是 | |
| 最大超时 | 60 秒 | |

**锁定字段**：`network`、`host_path_mode`、`read_only_rootfs`

即使 LLM 在工具调用中尝试传入 `network: 'on'` 或 `host_path_mode: 'rw'`，也会被忽略，始终使用 Profile 的安全值。

**适用场景**：
- 读取敏感配置文件
- 分析用户数据
- 只需要查看文件内容的场景

<Warning>
此 Profile 下，Agent 无法创建或修改任何文件。如果任务需要写入文件，请使用 `default` 或 `network_basic`。
</Warning>

---

### network_basic（基础网络）

需要网络访问时的推荐配置。

```yaml
box:
  local:
    profile: 'network_basic'
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 网络 | 开启 | 可以访问网络 |
| 文件挂载 | 读写 | |
| CPU 限制 | 1 核 | |
| 内存限制 | 512 MB | |
| PID 限制 | 128 | |
| 只读根文件系统 | 是 | |
| 最大超时 | 120 秒 | |

**适用场景**：
- 调用外部 API
- 下载文件
- 网络爬虫（简单）
- 安装 Python 包

---

### network_extended（扩展网络）

需要完全权限的网络任务。

```yaml
box:
  local:
    profile: 'network_extended'
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 网络 | 开启 | 完全网络访问 |
| 文件挂载 | 读写 | |
| CPU 限制 | 2 核 | |
| 内存限制 | 1024 MB | |
| PID 限制 | 256 | |
| 只读根文件系统 | 否 | 可以在根目录写入 |
| 最大超时 | 300 秒 | |

**适用场景**：
- 长时间运行的脚本
- 需要安装系统包的任务
- 复杂的网络操作
- 需要更多资源的任务

<Warning>
此 Profile 给予 Agent 更多权限，请确保信任您的 LLM 提供商。
</Warning>

## 资源限制详解

### CPU 限制

控制沙箱可使用的 CPU 时间：

| 值 | 说明 |
|-----|------|
| 0.5 | 最多使用 0.5 个 CPU 核心 |
| 1.0 | 最多使用 1 个 CPU 核心 |
| 2.0 | 最多使用 2 个 CPU 核心 |

### 内存限制

控制沙箱可使用的内存：

| 值 | 说明 |
|-----|------|
| 256 MB | 适合简单脚本 |
| 512 MB | 默认，适合大多数任务 |
| 1024 MB | 适合数据处理、机器学习 |

超出限制时，进程会被终止。

### PID 限制

控制沙箱内可创建的进程数：

| 值 | 说明 |
|-----|------|
| 64 | 最少，防止 fork 炸弹 |
| 128 | 默认 |
| 256 | 需要多进程任务 |

### 超时限制

单个命令的最大执行时间：

- Agent 可以指定 `timeout_sec`，但不能超过 Profile 的 `max_timeout_sec`
- 超时后命令会被强制终止

## 锁定字段

Profile 可以锁定某些字段，防止 Agent 通过工具参数覆盖：

```python
# Profile 定义中的 locked 字段
locked=frozenset({'network', 'host_path_mode', 'read_only_rootfs'})
```

当字段被锁定时：
- Agent 在调用 `exec` 工具时指定的参数会被忽略
- 始终使用 Profile 的默认值

这确保了即使 LLM 被诱导，也无法突破安全限制。

---

## 自定义配置

Profile 提供默认值，您可以在 `config.yaml` 中覆盖：

```yaml
box:
  local:
    profile: 'default'
    # 覆盖默认值
    workspace_quota_mb: 100    # 限制工作区大小为 100MB
```

### 工作区配额

限制沙箱工作区占用的磁盘空间：

```yaml
box:
  local:
    workspace_quota_mb: 500  # 500 MB
```

当工作区超过配额时：
- 新建 session 会被拒绝
- 现有 session 可能被销毁

## 安全最佳实践

### 1. 最小权限原则

根据实际需求选择权限最低的 Profile：

```
offline_readonly → default → network_basic → network_extended
   (最安全)                                    (最宽松)
```

### 2. 限制挂载目录

配置 `allowed_mount_roots` 限制 Agent 可访问的主机目录：

```yaml
box:
  local:
    allowed_mount_roots:
      - './data/box'
      - '/tmp'
```

Agent 只能挂载这些目录或其子目录。

### 3. 使用磁盘配额

防止 Agent 消耗过多磁盘空间：

```yaml
box:
  local:
    workspace_quota_mb: 100
```

### 4. 监控沙箱状态

通过 API 查看沙箱状态和错误：

```
GET /api/v1/box/status
GET /api/v1/box/errors
```
