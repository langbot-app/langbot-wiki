LangBot 支持使用来自 MCP 服务器的工具。您可以添加 MCP 服务器，并让支持 Tool Call 的模型调用这些工具处理请求。

<Info title="MCP 支持 Streamable HTTP、SSE 和 stdio 三种模式。">
  首选从 [LangBot Space](https://space.langbot.app) 安装：已上架的 MCP 服务器配置好、开箱即用。

  也可以从 [ModelScope](https://modelscope.cn/mcp)、[mcp.so](https://mcp.so)、[composio.dev](https://composio.dev/toolkits) 等渠道获取，再按下文手动添加。
</Info>

## 运行方式

| 模式 | 适合场景 | Runtime 关系 |
|------|----------|--------------|
| Streamable HTTP (`mode: "http"`) | 远程 MCP 服务，支持 Streamable HTTP 传输 | LangBot 直接连接远程 URL |
| SSE | 远程 MCP 服务，支持 SSE 传输 | LangBot 直接连接远程 URL |
| stdio | 本地命令启动的 MCP Server | 仅在 Box Runtime 可用时支持使用 |

<Note>
MCP 工具需要在流水线中使用 `内置 Agent`，并选择支持函数调用的模型。Dify、n8n、Langflow、Coze 等外部 Runner 使用各自平台的工具机制。
</Note>

## stdio MCP 与 Box Runtime

`stdio` 形式的 MCP 仅在 Box Runtime 可用时支持使用。满足条件后，LangBot 会在 Box 中启动并连接该 MCP 进程：

1. 创建或复用共享的 MCP Box session
2. 如果能从命令或参数推断本地项目路径，则把项目 staging 到共享工作区
3. 如果项目包含 `requirements.txt`、`pyproject.toml`、`setup.py` 或 `setup.cfg`，会先安装依赖
4. 通过 WebSocket relay 连接 MCP 进程的 stdin/stdout
5. MCP 工具列表成功加载后，工具才会进入 Agent 可用工具列表

高级配置中，stdio MCP 可在配置里包含 `box` 覆盖项：

```json
{
  "mode": "stdio",
  "extra_args": {
    "command": "python",
    "args": ["server.py"],
    "env": {},
    "box": {
      "host_path": "/path/to/mcp-project",
      "network": "on",
      "host_path_mode": "ro",
      "startup_timeout_sec": 120,
      "image": "python:3.11-slim"
    }
  }
}
```

常用字段：

| 字段 | 说明 |
|------|------|
| `host_path` | MCP 项目在宿主机上的目录。需要在 `box.local.allowed_mount_roots` 允许范围内 |
| `network` | 是否允许网络，常用于依赖安装 |
| `host_path_mode` | 准备 `host_path` 时的访问模式，默认 `ro`；stdio MCP 通常会先 staging 到共享工作区 |
| `startup_timeout_sec` | Box 准备和依赖安装超时，默认 120 秒；stdio MCP 整体连接等待会额外预留约 30 秒 |
| `image` | Docker 后端使用的运行镜像 |

<Warning>
stdio MCP 在 Box 中运行时，依赖安装和进程启动都会受到 sandbox profile、网络权限、资源限制和挂载白名单影响。
</Warning>

## 在流水线中启用

MCP Server 创建并连接后，还需要在流水线扩展配置中允许它：

- 开启“启用所有 MCP Server”
- 或绑定指定 MCP Server

如果绑定了指定列表，未绑定的 MCP Server 工具不会进入该流水线的 Agent 工具列表。

## 常见问题

| 现象 | 检查项 |
|------|--------|
| MCP 连接失败 | URL、命令、参数、环境变量是否正确 |
| stdio MCP 一直连接中 | Box Runtime 是否可用，`startup_timeout_sec` 是否过短 |
| 依赖安装失败 | sandbox 是否允许网络，项目中依赖文件是否正确 |
| 本地文件无法访问 | `host_path` 是否存在，是否在 `allowed_mount_roots` 内 |
| Agent 看不到 MCP 工具 | 流水线是否使用内置 Agent，模型是否开启函数调用，MCP 是否在流水线中绑定 |
