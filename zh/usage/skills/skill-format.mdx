---
title: "SKILL.md 格式"
---

每个技能是一个目录，根目录下放一个 `SKILL.md`。`SKILL.md` 包含**元数据 frontmatter** 和**指令正文**两部分。

## 整体结构

```markdown
---
name: my-skill
display_name: 我的技能
description: 简短描述，用于 Agent 判断何时激活
---

## 执行步骤

1. ...
2. ...
```

## Frontmatter 字段

| 字段 | 必需 | 说明 |
|------|------|------|
| `name` | 创建/注册时 | 唯一标识。1–64 字符，字母数字短横线下划线。直接扫描已存在的目录时可省略，会使用目录名 |
| `description` | 强烈建议 | 简短描述，决定 Agent 能否找到这个技能 |
| `display_name` | 否 | 显示名称（用于 UI） |

其它常见字段（`license` / `compatibility` / `metadata` / `allowed-tools`）当前不会被读取，可保留供未来兼容或人类阅读。

### 把 `description` 写好

`description` 是 Agent 唯一用来"挑技能"的依据——它会出现在 `<available_skills>` 列表里。描述模糊就等于这个技能不会被激活。

```yaml
# ❌ 太模糊
description: 帮助处理 PDF

# ✅ 包含功能、场景、触发关键词
description: 从 PDF 提取文本和表格，填充 PDF 表单，合并多个 PDF；用户提到 PDF、表单、文档提取时触发
```

写作要点：
- 说清楚**功能**（做什么）
- 说清楚**适用场景**（什么情况下用）
- 包含**触发关键词**（用户可能怎么说）

## 指令正文

Frontmatter 之后是 Markdown 正文。Agent 调用 `activate` 后，整个正文会作为 Tool Result 返回，注入到上下文。

建议结构（不强制）：

```markdown
## 使用场景
具体什么样的请求适合用这个技能。

## 执行步骤
1. ...
2. ...

## 可用资源
列出 scripts/、templates/ 下能用的文件，及其用途。

## 输出格式
如果输出有特定格式，在这里说明。
```

控制长度：正文建议在 500 行以内。超长的参考资料放到子目录里，按需用 `read` 工具加载。

## 完整示例

```markdown
---
name: api-tester
display_name: API 测试工具
description: 测试 REST API 端点，验证响应、跑性能基准；用户提到 API 测试、接口验证、压测时触发
---

## 使用场景

需要发起 HTTP 请求、校验响应、或对单个端点做性能压测时。

## 执行步骤

1. 从用户输入提取方法、URL、headers、body
2. 用 `exec` 调用 `scripts/request.py`
3. 校验状态码和响应体
4. 如需性能测试，调用 `scripts/benchmark.py`
5. 按模板输出结果

## 可用资源

- `scripts/request.py` — 单次 HTTP 请求
- `scripts/benchmark.py` — 并发压测
- `templates/test_report.md` — 报告模板

## 输出格式

单次：URL、状态码、响应时间、响应体摘要
批量：成功/失败数、平均响应时间、错误列表
```

## 目录布局

最小：

```
my-skill/
└── SKILL.md
```

带资源：

```
my-skill/
├── SKILL.md           # 必需
├── scripts/           # 可选：可执行脚本
├── templates/         # 可选：模板
└── requirements.txt   # 可选：依赖（Agent 可按需 pip install）
```

激活后，Agent 通过 `/workspace/.skills/{skill-name}/` 访问这些文件。

## 存放位置

沙箱后端可用时，技能存放在 Box Runtime 管理的 skill store，默认路径：

```
<box.local.host_root>/skills/<skill-name>/SKILL.md
```

`box.local.skills_root` 默认为相对路径 `skills`，会解析到 `host_root` 下。Box 不可用时技能列表为空，不会回退到其他目录。
