---
title: "技能工具 API"
---

Skills 系统给 Agent 提供两个工具。两个工具都需要沙箱后端可用——Box 不可用时不会暴露给 Agent。

| 工具 | 用途 |
|------|------|
| `activate` | 激活一个已有技能，把 SKILL.md 拉入上下文 |
| `register_skill` | 把沙箱里准备好的目录注册为新技能 |

---

## `activate`

### 参数

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `skill_name` | string | 是 | 要激活的技能名称 |

### 行为

1. 在 skill store 里查找该名称
2. 把 SKILL.md 内容作为 Tool Result 返回，注入到当前对话上下文
3. 该技能标记为已激活，可通过 `/workspace/.skills/{skill_name}/` 访问

激活后：
- `read` / `glob` / `grep` 可以读所有已挂载的技能
- 对技能包执行 `exec` / `write` / `edit` 仅限**已激活**的技能

### 调用示例

```json
{ "skill_name": "pdf" }
```

### 返回值

```json
{
  "activated": true,
  "skill_name": "pdf",
  "mount_path": "/workspace/.skills/pdf",
  "content": "<command-message>The \"pdf\" skill is activated</command-message>\n<skill-activation>\n  ...SKILL.md 内容...\n</skill-activation>"
}
```

### 错误

- `skill_name is required` — 未提供名称
- `Skill "xxx" not found` — 技能不存在

---

## `register_skill`

把 `/workspace` 下准备好的目录注册到 skill store。

### 参数

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `path` | string | 是 | 沙箱内目录路径，必须在 `/workspace` 下 |
| `name` | string | 否 | 覆盖技能名（默认用 SKILL.md 的 `name` 或目录名） |
| `display_name` | string | 否 | 覆盖显示名 |
| `description` | string | 否 | 覆盖描述 |
| `instructions` | string | 否 | 覆盖指令正文 |

### 前置条件

- 沙箱已启用且可用
- `path` 在 `/workspace` 下
- 该目录本身、或两层以内子目录里包含 `SKILL.md`

### 典型流程

```
1. Agent 在沙箱里准备目录：
   /workspace/my-skill/
   ├── SKILL.md
   └── scripts/main.py

2. Agent 调用 register_skill(path="/workspace/my-skill")

3. 技能写入 Box Runtime 的 skill store，
   之后任意会话都能通过 activate 使用
```

### 调用示例

```json
{
  "path": "/workspace/my-skill",
  "name": "data-processor",
  "description": "处理和转换 CSV/JSON 数据"
}
```

### 返回值

```json
{
  "registered": true,
  "skill_name": "data-processor",
  "source_path": "/workspace/my-skill",
  "skill": {
    "name": "data-processor",
    "display_name": "数据处理工具",
    "description": "处理和转换 CSV/JSON 数据",
    "package_root": "/.../skills/data-processor"
  }
}
```

### 错误

- `path is required` — 未提供路径
- `path must be under /workspace` — 路径不在工作区
- `Directory does not exist` — 目录不存在
- `No SKILL.md found in ...` — 目录及两层以内子目录没有 `SKILL.md`
- `Multiple skill directories found...` — 路径下有多个技能目录，请把 `path` 指向具体子目录
