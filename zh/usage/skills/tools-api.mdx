---
title: "技能工具 API"
---

Skills 系统为 Agent 提供了两个工具，用于激活和注册技能。设计遵循 Claude Code 的渐进式披露模式，通过 Tool Call 机制激活技能。

## 工具概览

| 工具 | 功能 | 需要沙箱 |
|------|------|---------|
| `activate` | 激活一个技能 | 是 |
| `register_skill` | 从沙箱目录注册技能 | 是 |

<Note>
这两个工具需要沙箱后端（Docker/Nsjail/E2B）可用才能使用。如果没有任何后端可用，工具将不会提供给 Agent。
</Note>

---

## activate

激活一个技能，将技能指令注入到当前对话上下文中。

### 使用场景

- 激活已有的技能以获取专业能力
- 获取技能的完整指令和运行时上下文

### 参数

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `skill_name` | string | 是 | 要激活的技能名称（无需参数） |

### 工作原理

1. Agent 调用 `activate` 工具，传入技能名称
2. 系统返回技能的 SKILL.md 内容作为 Tool Result
3. 技能包挂载到 `/workspace/.skills/{skill-name}/`
4. Agent 可使用 `exec`/`read`/`write`/`edit`/`glob`/`grep` 操作技能文件

### 示例

```json
{
  "skill_name": "pdf"
}
```

### 返回值

```json
{
  "activated": true,
  "skill_name": "pdf",
  "mount_path": "/workspace/.skills/pdf",
  "content": "<command-message>The \"pdf\" skill is activated</command-message>\n<skill-activation>\n<skill-name>pdf</skill-name>\n<mount-path>/workspace/.skills/pdf</mount-path>\n<package-root>/path/to/skill</package-root>\n\n## Instructions\n...\n\n## Runtime Context\nThe skill package is mounted at /workspace/.skills/pdf. Use the standard tools to interact with it:\n- Use `read` to inspect files under /workspace/.skills/pdf\n- Use `exec` with workdir set to /workspace/.skills/pdf to run commands in that package\n- Use `write` and `edit` on that path when the instructions require updating files\n</skill-activation>"
}
```

### 错误

- `skill_name is required`: 未提供技能名称
- `Skill "xxx" not found`: 技能不存在

## register_skill

从沙箱工作区目录注册技能包到 LangBot 技能存储。

### 使用场景

- 在沙箱中创建新技能后注册
- 导入预先准备的技能包

### 前提条件

- 沙箱功能已启用
- 目标目录存在于 `/workspace` 下
- 目录包含 `SKILL.md` 文件

### 参数

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `path` | string | 是 | 沙箱内的目录路径（必须在 `/workspace` 下，包含 SKILL.md） |
| `name` | string | 否 | 覆盖技能名称，默认使用 SKILL.md 中的名称或目录名 |
| `display_name` | string | 否 | 覆盖显示名称 |
| `description` | string | 否 | 覆盖描述 |
| `instructions` | string | 否 | 覆盖指令内容 |

### 工作流程

```
1. 在沙箱中准备技能目录
   /workspace/my-skill/
   ├── SKILL.md
   └── scripts/
       └── main.py

2. 调用 register_skill

3. 技能被注册到技能存储
   - 可通过 activate 工具激活使用
```

### 示例

```json
{
  "path": "/workspace/my-skill",
  "name": "data-processor",
  "description": "处理和转换数据文件"
}
```

### 返回值

```json
{
  "registered": true,
  "skill_name": "data-processor",
  "source_path": "/workspace/my-skill",
  "skill": {
    "name": "data-processor",
    "display_name": "数据处理工具",
    "description": "处理和转换数据文件",
    "instructions": "...",
    "package_root": "/path/to/data/skills/data-processor"
  }
}
```

### 错误

- `path is required`: 未提供路径
- `path must be under /workspace`: 路径不在工作区内
- `Directory does not exist`: 目录不存在
- `SKILL.md not found in directory`: 目录中没有 SKILL.md 文件

## 激活机制

Skills 系统采用 Tool Call 机制激活技能，而非文本标记。这与 Claude Code 的设计保持一致。

### 激活方式

Agent 通过调用 `activate` 工具来激活技能：

```json
{
  "skill_name": "pdf"
}
```

### 激活后效果

1. **指令注入**：SKILL.md 内容作为 Tool Result 返回，注入到上下文
2. **路径挂载**：技能包挂载到 `/workspace/.skills/{skill-name}/`
3. **文件操作**：Agent 可使用标准工具操作技能文件
   - `read`：读取技能文件
   - `exec`：在技能目录中执行命令
   - `write`/`edit`：修改技能文件

### 设计优势

| 特性 | 说明 |
|------|------|
| **KV Cache 保护** | 技能指令通过 Tool Result 按需加载，不影响对话历史的缓存效率 |
| **渐进式披露** | 工具描述中只显示技能元数据（名称、描述），完整指令在激活后获取 |
| **运行时隔离** | 每个激活的技能有独立的挂载路径，避免文件冲突 |

### 与文本标记的区别

| 方面 | 旧方案（文本标记） | 新方案（Tool Call） |
|------|------------------|-------------------|
| 激活方式 | `[ACTIVATE_SKILL: name]` | `activate` 工具调用 |
| 指令注入 | 预先注入 System Prompt | Tool Result 动态加载 |
| KV Cache | 影响 | 保护 |
| 多技能支持 | 多个标记 | 多次调用 |

## 创建新技能

LangBot 内置 `create-skill` 技能，用于引导创建新的技能包：

1. 激活 `create-skill` 技能
2. 按照指令准备技能文件
3. 使用 `register_skill` 工具注册技能
4. 新技能即可通过 `activate` 使用

<Note>
内置技能存放在 `templates/skills/` 目录，启动时自动复制到 `data/skills/`。
</Note>
