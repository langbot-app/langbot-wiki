---
title: "技能 (Skills)"
---

Skill 是一个**按需激活**的指令包：用户描述匹配时，Agent 调 `activate` 工具把这个技能的指令拉进上下文，然后按指令完成任务。

适合给 Agent 加一段**专业流程、模板或脚本**——比如"分析 Python 项目"、"做 PDF 表单填充"、"代码评审"——而不是每个对话都把所有指令塞进 System Prompt。

## 与插件、MCP 的区别

| 你想做的事 | 用这个 |
|------------|--------|
| 一段按需触发的工作流（含可选脚本/模板） | Skill |
| 接入已有的 MCP 工具服务 | MCP Server |
| 长期能力、配置页、事件监听、知识库 | 插件 |

详见[运行时与扩展关系](../sandbox/runtime-relationships)。

## 工作流程

```
用户："帮我分析这个 Python 项目"
   ↓
Agent 从 <available_skills> 看到 python-analyzer
   ↓
Agent 调用 activate(skill_name="python-analyzer")
   ↓
SKILL.md 内容作为 Tool Result 返回
   ↓
Agent 按指令执行（用 read / exec / write 等工具）
```

Agent 只在判断匹配时才激活技能，未激活的技能不占用上下文。

## 前置条件

Skills 依赖沙箱后端。`box.enabled` 必须为 `true` 且后端可用（Docker / Nsjail / E2B）。Box 不可用时，技能列表为空，新增/编辑/激活都不可用。

参见[沙箱配置](../sandbox/config)。

## 一个最小技能

技能就是一个目录加一个 `SKILL.md`：

```
my-skill/
└── SKILL.md
```

`SKILL.md`：

```markdown
---
name: code-reviewer
display_name: 代码评审助手
description: 审查代码并给出改进建议；用户提到 review、评审、代码改进时触发
---

## 执行步骤

1. 用 `read` 浏览相关文件
2. 检查可读性、潜在 bug、安全问题
3. 给出按优先级排序的改进建议
```

`description` 写得越清晰，Agent 越能准确判断何时激活——这是写好技能最重要的一件事。

详细字段说明和更多示例见 [SKILL.md 格式](./skill-format)。

## 创建技能

**方式一：WebUI** — Skills 页面创建、上传 zip、从 GitHub 导入

**方式二：沙箱 + `register_skill`**
1. 在沙箱 `/workspace` 下准备好目录和 `SKILL.md`
2. 让 Agent 调用 `register_skill(path="/workspace/my-skill")`

两种方式都会写入 Box Runtime 管理的 skill store。

## 在流水线启用

技能装好之后，还要在流水线扩展配置里决定 Agent 能看到哪些：

- 开启"启用所有技能"，或
- 只绑定指定技能

绑定只影响 `<available_skills>` 列表（即 Agent 能选什么）。完整指令永远等 `activate` 被调用后才注入。

## Agent 可用的工具

| 工具 | 用途 |
|------|------|
| `activate` | 激活一个技能，把 SKILL.md 拉进上下文 |
| `register_skill` | 把沙箱里准备好的目录注册为技能 |

参数和返回值见[工具 API](./tools-api)。

## 常见问题

| 现象 | 检查项 |
|------|--------|
| 技能列表为空 | `box.enabled`、Box Runtime 状态、`box.local.skills_root` |
| Agent 不激活技能 | `description` 是否具体、流水线是否绑定、模型是否支持函数调用 |
| 激活后找不到文件 | 文件需在 Skill 包内，路径是 `/workspace/.skills/{skill-name}/` |
| `register_skill` 报错 | 路径必须在 `/workspace` 下，目录或两层内子目录需含 `SKILL.md` |

<Note>
Skills 当前面向内置 Agent，依赖函数调用和沙箱后端。外部 Runner 请使用各自平台的等价机制。
</Note>
