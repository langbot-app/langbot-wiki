---
title: "技能系统 (Skills)"
---

Skills（技能）是 LangBot 的动态技能系统，允许 AI Agent 通过 Tool Call 机制按需激活和执行特定任务的能力。通过 Skills，您可以让 Agent 在需要时自动加载专业领域的知识和工具。

## 什么是 Skills？

Skills 是一种轻量级的扩展机制，让 Agent 能够：

- 通过 Tool Call 按需加载专业领域知识
- 执行特定任务的工作流
- 访问预定义的代码和资源
- 动态组合多个技能

与插件不同，Skills 更轻量、更灵活，适合定义特定任务的工作流和指令。

## 设计理念

### 业界实践

Skills 系统的设计参考了业界主流的 Agent 扩展模式：

| 系统 | 机制 | 特点 |
|------|------|------|
| **Claude Code** | Skills + Commands | Tool Call 激活，Tool Result 注入（保护 KV Cache） |
| **OpenAI GPTs** | Actions + Knowledge | 分离工具调用和知识注入 |
| **Cursor** | Rules + Tools | 规则系统和工具系统独立配置 |

### Claude Code 的设计启示

Claude Code 的 Skills 系统有一个关键设计决策：**技能内容通过 Tool Result 注入，而非直接修改 System Prompt**。

这种设计的原因是：

1. **保护 KV Cache**：直接修改 System Prompt 会导致之前的 KV Cache 失效，每次都需要重新计算
2. **按需加载**：技能只有在 Agent 主动调用 `activate` 工具时才会被注入，减少不必要的 token 消耗
3. **增量上下文**：Tool Result 是增量内容，不会影响已有的缓存

### LangBot 的设计

LangBot 完全采用了 Claude Code 的设计模式：

```
┌─────────────────────────────────────────┐
│         Agent (LLM)                      │
│                                          │
│  用户请求："帮我分析这个 Python 项目"     │
│                                          │
│  判断：匹配 skill: python-analyzer       │
│                                          │
│  调用：activate(skill_name="python-...") │
│                                          │
└───────────────────┬─────────────────────┘
                    │ Tool Call
                    ▼
┌─────────────────────────────────────────┐
│           activate 工具                  │
│                                          │
│  返回：Tool Result                       │
│  - SKILL.md 内容                         │
│ - 挂载路径信息                           │
│                                          │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│      Agent 继续执行（带技能上下文）        │
│                                          │
│  - 使用 exec/read/write/edit 操作技能包  │
│  - 按照技能指令执行任务                   │
│  - 可多次调用 activate 激活多个技能       │
│                                          │
└─────────────────────────────────────────┘
```

### 为什么这样设计？

**1. KV Cache 保护**

技能指令通过 Tool Result 返回，是增量内容，不会使已有的 KV Cache 失效。

**2. 渐进式披露**

- `activate` 工具描述中只显示技能元数据（名称、描述）
- 完整指令在激活后通过 Tool Result 获取
- Agent 可以根据需要选择是否激活

**3. 统一的交互模式**

- 与其他工具调用语义一致
- 更符合 Agent 设计模式
- 对支持 Function Calling 的模型更友好

**4. 需要沙箱后端**

`activate` 和 `register_skill` 工具需要沙箱后端可用：
- 技能包挂载到 `/workspace/.skills/{skill-name}/`
- Agent 使用 `exec`/`read`/`write`/`edit`/`glob`/`grep` 操作技能文件

## Skill 文件结构

每个 Skill 是一个包含 `SKILL.md` 文件的目录：

```
data/skills/
├── my-skill/
│   ├── SKILL.md           # 必需：技能定义文件
│   ├── scripts/           # 可选：脚本文件
│   │   └── analyze.py
│   └── templates/         # 可选：模板文件
│       └── report.md
└── another-skill/
    └── SKILL.md
```

### SKILL.md 格式

使用 YAML frontmatter 定义元数据：

```markdown
---
name: python-analyzer
display_name: Python 项目分析器
description: 分析 Python 项目结构、代码质量和依赖关系
---

## 使用场景

当用户需要分析 Python 项目时，按以下步骤执行：

1. 使用 `read` 工具读取项目结构
2. 使用 `exec` 运行分析脚本
3. 生成分析报告

## 可用脚本

- `scripts/analyze.py`: 主分析脚本
- `scripts/dependency_check.py`: 依赖检查

## 输出格式

分析报告应包含：
- 项目结构概览
- 代码质量评分
- 依赖关系图
```

## 激活机制

### Tool Call 激活

Agent 通过调用 `activate` 工具来激活技能：

```json
{
  "skill_name": "python-analyzer"
}
```

### 激活后效果

1. **Tool Result 注入**：SKILL.md 内容作为 Tool Result 返回，注入到上下文
2. **挂载路径**：技能包挂载到 `/workspace/.skills/{skill-name}/`
3. **工具访问**：Agent 可以通过 `exec`/`read`/`write`/`edit`/`glob`/`grep` 操作技能文件

### 返回内容示例

```
<command-message>The "python-analyzer" skill is activated</command-message>
<skill-activation>
<skill-name>python-analyzer</skill-name>
<mount-path>/workspace/.skills/python-analyzer</mount-path>
<package-root>/path/to/skill</package-root>

## Instructions
...SKILL.md 内容...

## Runtime Context
The skill package is mounted at /workspace/.skills/python-analyzer. Use the standard tools to interact with it:
- Use `read` to inspect files under /workspace/.skills/python-analyzer
- Use `exec` with workdir set to /workspace/.skills/python-analyzer to run commands
- Use `write` and `edit` on that path when the instructions require updating files
</skill-activation>
```

## 内置工具

Skills 系统提供两个工具供 Agent 使用：

| 工具 | 功能 | 需要沙箱 |
|------|------|---------|
| `activate` | 激活一个技能 | 是 |
| `register_skill` | 从沙箱目录注册技能 | 是 |

<Note>
这两个工具需要沙箱后端（Docker/Nsjail/E2B）可用。如果没有任何后端可用，工具将不会提供给 Agent。
</Note>

## 技能类型

### 纯提示词技能

只包含指令，不需要额外文件：

```markdown
---
name: code-reviewer
display_name: 代码审查助手
description: 审查代码并提供改进建议
---

## 审查步骤

1. 分析代码结构和风格
2. 检查潜在的 bug 和安全问题
3. 提供改进建议

## 审查标准

- 代码可读性
- 性能优化
- 安全最佳实践
```

### 文件包技能

包含脚本、模板等资源文件：

```
data-analyzer/
├── SKILL.md
├── scripts/
│   ├── analyze.py
│   └── visualize.py
├── templates/
│   └── report.md
└── requirements.txt
```

## 快速开始

### 1. 创建第一个技能

在 `data/skills/` 目录下创建技能：

```bash
mkdir -p data/skills/my-first-skill
```

创建 `SKILL.md`：

```markdown
---
name: my-first-skill
display_name: 我的第一个技能
description: 一个简单的示例技能
---

## 任务说明

这是一个示例技能，演示基本的技能功能。

## 执行步骤

1. 理解用户请求
2. 执行相应操作
3. 返回结果
```

### 2. 测试激活

确保沙箱后端可用，然后向 Agent 发送匹配技能描述的请求，Agent 会自动调用 `activate` 工具激活技能。

## 创建新技能

### 方式一：使用内置 create-skill 技能

LangBot 内置了 `create-skill` 技能，用于引导创建新的技能包：

1. 调用 `activate(skill_name="create-skill")`
2. 按照返回的指令准备技能文件
3. 使用 `register_skill` 工具注册技能

### 方式二：手动创建

1. 在沙箱 `/workspace` 下创建技能目录
2. 编写 `SKILL.md` 和相关文件
3. 调用 `register_skill(path="/workspace/my-skill")` 注册

## 内置技能

LangBot 启动时会自动从 `templates/skills/` 复制内置技能到 `data/skills/`：

| 技能 | 描述 |
|------|------|
| `create-skill` | 引导创建新技能的技能 |

## 最佳实践

### 1. 技能命名

- 使用小写字母、数字、连字符、下划线
- 名称应简洁且具有描述性
- 示例：`python-analyzer`、`data-visualizer`

### 2. 描述编写

- 清晰说明技能的用途和适用场景
- 帮助 LLM 正确判断何时激活
- 描述会显示在 `activate` 工具的 `<available_skills>` 列表中

### 3. 指令设计

- 提供清晰的执行步骤
- 说明可用的资源和工具
- 定义输出格式

### 4. 资源组织

- 脚本放在 `scripts/` 目录
- 模板放在 `templates/` 目录
- 保持结构清晰
