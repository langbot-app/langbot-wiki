Complete sandbox configuration options.

## Configuration Location

Sandbox configuration is in the `box` section of `data/config.yaml`:

```yaml
box:
  enabled: true
  backend: 'local'
  runtime:
    endpoint: ''
  local:
    profile: 'default'
    image: ''
    host_root: './data/box'
    default_workspace: ''
    skills_root: 'skills'
    allowed_mount_roots:
      - './data/box'
      - '/tmp'
    workspace_quota_mb: null
  e2b:
    api_key: ''
    api_url: ''
    template: ''
```

## Top-Level Configuration

### `enabled`

Master switch for the sandbox runtime.

| Value | Description |
|-------|-------------|
| `true` | Enable the sandbox (default) |
| `false` | Fully disable the sandbox |

Default: `true`

When set to `false`, LangBot does not connect to a remote Box Runtime and does not start a local stdio Box subprocess. All sandbox-dependent features turn off as a result: the built-in tools (`exec`/`read`/`write`/`edit`/`glob`/`grep`), the `activate` skill tool, skill creation/editing, and stdio-mode MCP servers. Skills can still be listed read-only, and http/sse-mode MCP servers are unaffected.

### `backend`

Force a specific sandbox backend type.

| Value | Description |
|-------|-------------|
| Empty/Not set | Try all backends (Docker → Nsjail → E2B), use first available |
| `local` | Only try local backends (Docker → Nsjail), don't try E2B |
| `docker` | Force Docker only |
| `nsjail` | Force Nsjail only |
| `e2b` | Force E2B cloud sandbox only |

Default: `local`

<Note>
This is a **forced value**, not a priority list. For example, when set to `local`, E2B won't be used even if available.
</Note>

### `runtime.endpoint`

External Box Runtime WebSocket URL.

| Value | Description |
|-------|-------------|
| Empty | Auto-detect (stdio locally, WebSocket in container) |
| `ws://host:port` | Connect to specified Runtime |

Default: Empty

<Note>
Most cases don't need this configured. Only use when deploying Box Runtime independently.
</Note>

## local Configuration

### `local.profile`

Security profile name.

| Value | Description |
|-------|-------------|
| `default` | Default profile, no network, read/write mount |
| `offline_readonly` | High security, no network, read-only mount |
| `network_basic` | Network enabled, read/write mount |
| `network_extended` | Full network, read/write mount, more resources |

Default: `default`

See [Security Profiles](./profiles) for details.

### `local.image`

Custom sandbox image.

| Value | Description |
|-------|-------------|
| Empty | Use default image `rockchin/langbot-sandbox:latest` |
| `python:3.11-slim` | Use Python 3.11 image |
| `node:20-slim` | Use Node.js 20 image |

Default: Empty

<Note>
Only applies to Docker backend. Nsjail uses host system environment, doesn't support custom images.
</Note>

### `local.host_root`

Host sandbox working directory root path.

Sandbox workspace files will be stored under this directory.

| Value | Description |
|-------|-------------|
| `./data/box` | Relative to the LangBot process working directory |
| `/var/lib/langbot/box` | Absolute path |

Default: `./data/box`

### `local.default_workspace`

Default workspace name.

| Value | Description |
|-------|-------------|
| Empty | Use `<host_root>/default` |
| `my-workspace` | Use `<host_root>/my-workspace` |

Default: Empty

### `local.skills_root`

Box Runtime-owned Skill package directory.

| Value | Description |
|-------|-------------|
| `skills` | Use `<host_root>/skills` |
| `/var/lib/langbot/skills` | Use an absolute path |

Default: `skills`

<Note>
Skills are entirely owned by the Box Runtime, which is the sole source of skills. When Box is unavailable (disabled via configuration or a failed connection), the skill list is simply empty — there is no local fallback that reads from `data/skills`.
</Note>

### `local.allowed_mount_roots`

Whitelist of host directories Agent can mount.

Agent's requested `host_path` must be under one of the directories in this list.

```yaml
allowed_mount_roots:
  - './data/box'     # Allow mounting sandbox working directory
  - '/tmp'           # Allow mounting temp directory
  - '/home/user/projects'  # Allow mounting project directory
```

Default: `['<host_root>']` (only allow mounting sandbox working directory)

### `local.workspace_quota_mb`

Workspace disk quota in MB.

| Value | Description |
|-------|-------------|
| `null` | Use Profile default (usually 0, unlimited) |
| `0` | No limit |
| `100` | Limit to 100 MB |
| `500` | Limit to 500 MB |

Default: `null`

<Note>
This quota is enforced. Existing sessions may be destroyed when exceeded.
</Note>

## e2b Configuration

### `e2b.api_key`

E2B API Key.

Can also be set via `E2B_API_KEY` environment variable, which takes precedence.

| Source | Priority |
|--------|----------|
| `E2B_API_KEY` env var | High |
| `e2b.api_key` in config.yaml | Low |

### `e2b.api_url`

Custom E2B API URL.

For connecting to self-hosted E2B service. Can also be set via `E2B_API_URL` environment variable.

| Value | Description |
|-------|-------------|
| Empty | Use official E2B service |
| `https://your-e2b.example.com` | Self-hosted service address |

### `e2b.template`

Default E2B template ID.

| Value | Description |
|-------|-------------|
| Empty | Use E2B default template |
| `base` | Base template |
| `python-3.11` | Python 3.11 template |
| Custom ID | Your custom template created in E2B |

## Environment Variables

LangBot uses a unified environment-variable override convention: `SECTION__SUBSECTION__KEY` (separated by double underscores) overrides the matching field in `config.yaml`. The `box` configuration can be set this way:

| Environment Variable | Config Item |
|---------------------|-------------|
| `BOX__BACKEND` | `box.backend` |
| `BOX__LOCAL__PROFILE` | `box.local.profile` |
| `BOX__LOCAL__IMAGE` | `box.local.image` |
| `BOX__LOCAL__HOST_ROOT` | `box.local.host_root` |
| `BOX__LOCAL__DEFAULT_WORKSPACE` | `box.local.default_workspace` |
| `BOX__LOCAL__SKILLS_ROOT` | `box.local.skills_root` |
| `BOX__LOCAL__ALLOWED_MOUNT_ROOTS` | `box.local.allowed_mount_roots`, comma-separated |
| `BOX__LOCAL__WORKSPACE_QUOTA_MB` | `box.local.workspace_quota_mb` |
| `BOX__E2B__API_KEY` | `box.e2b.api_key` |
| `BOX__E2B__API_URL` | `box.e2b.api_url` |
| `BOX__E2B__TEMPLATE` | `box.e2b.template` |
| `BOX_BACKEND` | Forces the Box Runtime backend and takes precedence over `box.backend` |

<Note>
E2B SDK also supports `E2B_API_KEY` and `E2B_API_URL` environment variables, which take precedence over `config.yaml`.

The Box Runtime process (the `langbot_box` service in containerized deployments) **does not read** `config.yaml` or environment variables — its configuration is pushed by the LangBot main process over INIT RPC. The `BOX__*` and similar environment variables therefore only need to be set on the `langbot` service.
</Note>

## Docker Compose Example

With Docker Compose, run the Box Runtime as a separate `langbot_box` service. It is an **optional service** gated behind a compose profile — it only starts when you run `docker compose --profile box up` (or `--profile all`). `langbot_box` shares the same host Box root with `langbot`:

```yaml
# docker-compose.yaml
services:
  langbot_box:
    image: rockchin/langbot:latest
    # Optional service: whether it starts is controlled by a compose profile
    profiles: ["box", "all"]
    volumes:
      # Keep source and target identical because langbot_box creates
      # sandbox containers through the host Docker socket.
      - ${LANGBOT_BOX_ROOT:-${PWD}/data/box}:${LANGBOT_BOX_ROOT:-${PWD}/data/box}
      - /var/run/docker.sock:/var/run/docker.sock
    # langbot_box does not read config.yaml or environment variables; its
    # config is pushed by langbot over INIT RPC. Do not add BOX__* /
    # LANGBOT_BOX_* variables here — they would be ignored.
    command: ["uv", "run", "--no-sync", "-m", "langbot_plugin.cli.__init__", "box"]

  langbot:
    image: rockchin/langbot:latest
    volumes:
      - ./data:/app/data
    environment:
      # Unified environment-variable override convention; these box.local.*
      # settings are forwarded to the Box Runtime by langbot over INIT RPC.
      - BOX__LOCAL__HOST_ROOT=${LANGBOT_BOX_ROOT:-${PWD}/data/box}
      - BOX__LOCAL__DEFAULT_WORKSPACE=default
      - BOX__LOCAL__SKILLS_ROOT=skills
      - BOX__LOCAL__ALLOWED_MOUNT_ROOTS=${LANGBOT_BOX_ROOT:-${PWD}/data/box}
```

<Warning>
`langbot_box` needs access to the Docker daemon. Mount `docker.sock` only in trusted environments. Because `langbot_box` uses the host Docker socket to create sandbox containers, the host path and container path for the Box root must be identical.
</Warning>

## Complete Configuration Examples

### Basic Configuration

```yaml
box:
  backend: 'local'
  local:
    profile: 'default'
    host_root: './data/box'
```

### Development Environment

```yaml
box:
  backend: 'local'
  local:
    profile: 'network_extended'
    host_root: './data/box'
    allowed_mount_roots:
      - './data/box'
      - '/tmp'
      - './projects'
```

### High Security Configuration

```yaml
box:
  backend: 'local'
  local:
    profile: 'offline_readonly'
    host_root: './data/box'
    allowed_mount_roots:
      - './data/box'
    workspace_quota_mb: 50
```

### E2B Cloud Sandbox

```yaml
box:
  backend: 'e2b'
  e2b:
    api_key: 'e2b_xxxxx'
    template: 'python-3.11'
```
