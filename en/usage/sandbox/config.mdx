---
title: "Sandbox Configuration"
---

Sandbox settings are owned by the LangBot main process. Box Runtime receives the effective configuration and performs the actual execution. In most deployments, you only need to choose a backend, set the workspace directory, and pick a security profile.

## Recommended Configuration

```yaml
box:
  enabled: true
  backend: 'local'
  local:
    profile: 'default'
    host_root: './data/box'
    skills_root: 'skills'
```

- `enabled`: master switch. When set to `false`, sandbox tools, Skill add/edit, stdio MCP hosting, and other Box-dependent features are unavailable.
- `backend`: backend selection. The default `local` only tries local Docker/Nsjail backends.
- `local.profile`: security profile that controls network, mounts, and resource limits.
- `local.host_root`: host workspace root, mapped to `/workspace` inside the sandbox.
- `local.skills_root`: Skill package directory. Relative paths resolve under `host_root`; the default is `host_root/skills`.

<Note>
In the current implementation, Skills are loaded only from the Box Runtime-managed skill store. When Box Runtime or a backend is unavailable, the Skill list is empty and create/import/edit/`register_skill` are unavailable; LangBot does not fall back to scanning `data/skills/`.
</Note>

## Backend Selection

| Backend | Use Case | Notes |
|---------|----------|-------|
| `local` | Recommended default | Tries Docker, then Nsjail; does not try E2B |
| `docker` | Production or container isolation | Requires Docker daemon; supports custom images |
| `nsjail` | Lightweight Linux isolation | Requires `nsjail`; no custom image support |
| `e2b` | Cloud sandbox | Requires an E2B API key; no local container runtime needed |

`backend` is a forced value, not a priority list. For example, `local` will not use E2B even if E2B is configured.

## Security Profiles

| Profile | Network | Mounts | Resources | Recommended For |
|---------|---------|--------|-----------|-----------------|
| `default` | Off | Read-write | Default limits | Default choice |
| `offline_readonly` | Off | Read-only | Stricter | Reading untrusted files |
| `network_basic` | Basic network | Read-write | Default limits | API access or dependency downloads |
| `network_extended` | Full network | Read-write | Looser | Development, debugging, complex tasks |

Prefer least privilege: use `default` or `offline_readonly` when network access is unnecessary, and add only required directories to `allowed_mount_roots`.

## Common Options

| Option | Default | Description |
|--------|---------|-------------|
| `runtime.endpoint` | empty | External Box Runtime WebSocket URL; usually not needed |
| `local.image` | empty | Custom Docker image; empty uses the default image |
| `local.default_workspace` | empty | Default workspace name; empty uses `host_root/default` |
| `local.allowed_mount_roots` | `host_root` | Host directories the Agent may request to mount |
| `local.workspace_quota_mb` | `null` | Workspace disk quota; `null` uses the profile default |
| `e2b.api_key` | empty | E2B API key; can also be set with `E2B_API_KEY` |
| `e2b.api_url` | empty | Self-hosted E2B endpoint; can also be set with `E2B_API_URL` |
| `e2b.template` | empty | Default E2B template ID |

## Docker Compose

With Docker Compose, set Box configuration on the `langbot` service. LangBot sends the effective configuration to `langbot_box` through INIT RPC.

```yaml
services:
  langbot_box:
    image: rockchin/langbot:latest
    container_name: langbot_box
    profiles: ["box", "all"]
    volumes:
      - ${LANGBOT_BOX_ROOT:-${PWD}/data/box}:${LANGBOT_BOX_ROOT:-${PWD}/data/box}
      - /var/run/docker.sock:/var/run/docker.sock
    command: ["uv", "run", "--no-sync", "-m", "langbot_plugin.cli.__init__", "box"]

  langbot:
    image: rockchin/langbot:latest
    volumes:
      - ./data:/app/data
    environment:
      - BOX__LOCAL__HOST_ROOT=${LANGBOT_BOX_ROOT:-${PWD}/data/box}
      - BOX__LOCAL__SKILLS_ROOT=skills
      - BOX__LOCAL__ALLOWED_MOUNT_ROOTS=${LANGBOT_BOX_ROOT:-${PWD}/data/box}
```

<Warning>
`langbot_box` needs access to Docker daemon. Mount `docker.sock` only in trusted environments, and keep the Box root path identical on the host and inside the `langbot_box` container.
</Warning>

## Environment Variables

| Environment Variable | Writes To |
|----------------------|-----------|
| `BOX__ENABLED` | `box.enabled` |
| `BOX__BACKEND` | `box.backend` |
| `BOX__LOCAL__PROFILE` | `box.local.profile` |
| `BOX__LOCAL__IMAGE` | `box.local.image` |
| `BOX__LOCAL__HOST_ROOT` | `box.local.host_root` |
| `BOX__LOCAL__DEFAULT_WORKSPACE` | `box.local.default_workspace` |
| `BOX__LOCAL__SKILLS_ROOT` | `box.local.skills_root` |
| `BOX__LOCAL__ALLOWED_MOUNT_ROOTS` | `box.local.allowed_mount_roots`, comma-separated |
| `BOX__LOCAL__WORKSPACE_QUOTA_MB` | `box.local.workspace_quota_mb` |
| `BOX__E2B__API_KEY` | `box.e2b.api_key` |
| `BOX__E2B__API_URL` | `box.e2b.api_url` |
| `BOX__E2B__TEMPLATE` | `box.e2b.template` |
| `BOX_BACKEND` | Box Runtime-level backend override |

<Note>
Do not set `LANGBOT_BOX_*` or `BOX__*` on the `langbot_box` service; Box Runtime does not read those variables directly.
</Note>
