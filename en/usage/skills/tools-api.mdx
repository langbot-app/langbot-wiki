title: "Skill Tools API"
---

The Skills system provides a series of tools for Agents to create, manage, and use skills.

## Tools Overview

| Tool | Function | Needs Sandbox |
|------|----------|---------------|
| `create_skill` | Create new managed skill | No |
| `list_skills` | List all managed skills | No |
| `get_skill` | Get single skill details | No |
| `update_skill` | Update existing skill | No |
| `delete_skill` | Delete skill | No |
| `import_skill_from_directory` | Import skill from workspace | Yes |
| `reload_skills` | Reload skills | No |

---

## create_skill

Create a new managed skill.

### Use Cases

- Create prompt-only skills
- Quickly define new task capabilities
- Skills that don't need additional file resources

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | Yes | Skill name (unique identifier) |
| `instructions` | string | Yes | SKILL.md body instructions |
| `display_name` | string | No | Display name |
| `description` | string | No | Skill description |

### Example

```json
{
  "name": "code-formatter",
  "display_name": "Code Formatter",
  "description": "Format code in various programming languages",
  "instructions": "## Usage\n\n1. Identify code language\n2. Apply corresponding formatting rules\n3. Return formatted result\n\n## Supported Languages\n\n- Python (black)\n- JavaScript (prettier)\n- Go (gofmt)"
}
```

### Return Value

```json
{
  "created": true,
  "skill": {
    "name": "code-formatter",
    "display_name": "Code Formatter",
    "description": "Format code in various programming languages",
    "instructions": "..."
  }
}
```

### Errors

- `name is required`: Skill name not provided
- `instructions is required`: Instructions not provided
- Skill name already exists

- Confirm skill instructions
- Check before preparing update

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | Yes | Skill name |

### Example

```json
{

}
```

### Return Value

```json

  "skill": {
    "name": "python-analyzer",
    "display_name": "Python Project Analyzer",
    "description": "Analyze Python project structure",
    "instructions": "## Use Cases\n...",

```

- `name is required`: Skill name not provided
- `Skill "xxx" not found`: Skill does not exist

## import_skill_from_directory

Import a skill package from sandbox workspace.

### Use Cases

- Import skills containing script files
- Import skill repositories cloned from Git
- Import pre-prepared complex skills

### Prerequisites

- Sandbox feature is enabled
- Target directory exists under `/workspace`
- Directory contains `SKILL.md` file

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Directory path in sandbox (must be under `/workspace`) |
| `name` | string | No | Override skill name |
| `display_name` | string | No | Override display name |
| `description` | string | No | Override description |
| `instructions` | string | No | Override instruction content |

### Workflow

```
1. Prepare skill directory in sandbox
   /workspace/my-skill/
   ├── SKILL.md
   └── scripts/
       └── main.py

2. Call import_skill_from_directory

3. Skill is imported to managed skill storage
   data/skills/my-skill/
```

### Example

```json
{
  "path": "/workspace/my-skill",
  "name": "data-processor",
  "description": "Process and transform data files"
}
```

### Return Value

```json
{
  "imported": true,
  "source_path": "/workspace/my-skill",
  "skill": {
    "name": "data-processor",
    "display_name": "Data Processing Tool",
    "description": "Process and transform data files",
    "instructions": "...",
    "package_root": "/path/to/data/skills/data-processor"
  }
}
```

### Errors

- `path is required`: Path not provided
- `path must be under /workspace`: Path not in workspace
- `Directory does not exist`: Directory does not exist
- `No default workspace configured`: Sandbox not configured

## Activation Mechanism

Besides the management tools above, the Skills system also provides an activation mechanism.

### Activation Marker

LLM activates skills by outputting a special marker at the beginning of its response:

```
[ACTIVATE_SKILL: skill-name]
```

### Multi-Skill Activation

Multiple skills can be activated simultaneously:

```
[ACTIVATE_SKILL: python-analyzer]
[ACTIVATE_SKILL: report-generator]
```

### Effects After Activation

1. Skill instructions are injected into System Prompt
2. Skill package is mounted to `/workspace/.skills/{skill-name}/`
3. Agent can use `exec`/`read`/`write`/`edit` to operate skill files

### Limitations

- Only skills bound to the current Pipeline can be activated
- Activation occurs during response generation, requiring multi-turn dialogue
- After skill activation, the original marker is removed from the response
