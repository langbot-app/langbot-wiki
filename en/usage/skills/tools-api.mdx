The Skills system provides two tools for Agents to activate and register skills. The design follows Claude Code's progressive disclosure pattern, using Tool Call mechanism for skill activation.

## Tools Overview

| Tool | Function | Needs Sandbox |
|------|----------|---------------|
| `activate` | Activate a skill | Yes |
| `register_skill` | Register skill from sandbox directory | Yes |

<Note>
These tools require a sandbox backend (Docker/Nsjail/E2B) to be available. If no backend is available, the tools will not be provided to the Agent.
</Note>

---

## activate

Activate a skill, injecting the skill instructions into the current conversation context.

### Use Cases

- Activate existing skills to gain specialized capabilities
- Get full instructions and runtime context of a skill

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `skill_name` | string | Yes | The skill name to activate (no arguments) |

### How It Works

1. Agent calls the `activate` tool with the skill name
2. System returns the skill's SKILL.md content as Tool Result
3. Skill package is mounted to `/workspace/.skills/{skill-name}/`
4. Agent can use `exec`/`read`/`write`/`edit`/`glob`/`grep` to operate skill files

### Example

```json
{
  "skill_name": "pdf"
}
```

### Return Value

```json
{
  "activated": true,
  "skill_name": "pdf",
  "mount_path": "/workspace/.skills/pdf",
  "content": "<command-message>The \"pdf\" skill is activated</command-message>\n<skill-activation>\n<skill-name>pdf</skill-name>\n<mount-path>/workspace/.skills/pdf</mount-path>\n<package-root>/path/to/skill</package-root>\n\n## Instructions\n...\n\n## Runtime Context\nThe skill package is mounted at /workspace/.skills/pdf. Use the standard tools to interact with it:\n- Use `read` to inspect files under /workspace/.skills/pdf\n- Use `exec` with workdir set to /workspace/.skills/pdf to run commands in that package\n- Use `write` and `edit` on that path when the instructions require updating files\n</skill-activation>"
}
```

### Errors

- `skill_name is required`: Skill name not provided
- `Skill "xxx" not found`: Skill does not exist

## register_skill

Register a skill package from sandbox workspace directory to LangBot skill store.

### Use Cases

- Register a skill after creating it in the sandbox
- Import pre-prepared skill packages

### Prerequisites

- Sandbox feature is enabled
- Target directory exists under `/workspace`
- Directory contains `SKILL.md` file

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Directory path in sandbox (must be under `/workspace`, containing SKILL.md) |
| `name` | string | No | Override skill name, defaults to name in SKILL.md or directory name |
| `display_name` | string | No | Override display name |
| `description` | string | No | Override description |
| `instructions` | string | No | Override instruction content |

### Workflow

```
   â””â”€â”€ scripts/
       â””â”€â”€ main.py

2. Call register_skill

3. Skill is registered to skill store

```

### Example

```json
{
  "path": "/workspace/my-skill",
  "name": "data-processor",
  "description": "Process and transform data files"
}
```

### Return Value

```json
{
  "registered": true,
  "skill_name": "data-processor",
  "source_path": "/workspace/my-skill",
  "skill": {
    "name": "data-processor",
```

- `path is required`: Path not provided
- `path must be under /workspace`: Path not in workspace
- `Directory does not exist`: Directory does not exist
- `SKILL.md not found in directory`: No SKILL.md file in directory

## Activation Mechanism

The Skills system uses Tool Call mechanism for skill activation, not text markers. This aligns with Claude Code's design.

### Activation Method

Agent activates a skill by calling the `activate` tool:

```json
{
  "skill_name": "pdf"
}
```

### Effects After Activation

1. **Instruction Injection**: SKILL.md content is returned as Tool Result, injected into context
2. **Path Mounting**: Skill package is mounted to `/workspace/.skills/{skill-name}/`
3. **File Operations**: Agent can use standard tools to operate skill files
   - `read`: Read skill files
   - `exec`: Execute commands in skill directory
   - `write`/`edit`: Modify skill files

### Design Advantages

| Feature | Description |
|---------|-------------|
| **KV Cache Protection** | Skill instructions are loaded on-demand via Tool Result, not affecting conversation history cache efficiency |
| **Progressive Disclosure** | Tool description only shows skill metadata (name, description), full instructions obtained after activation |
| **Runtime Isolation** | Each activated skill has independent mount path, avoiding file conflicts |

### Comparison with Text Markers

| Aspect | Old Approach (Text Marker) | New Approach (Tool Call) |
|--------|---------------------------|--------------------------|
| Activation Method | `[ACTIVATE_SKILL: name]` | `activate` tool call |
| Instruction Injection | Pre-injected into System Prompt | Dynamically loaded via Tool Result |
| KV Cache | Affected | Protected |
| Multi-Skill Support | Multiple markers | Multiple calls |

## Creating New Skills

LangBot includes a built-in `create-skill` skill for guiding the creation of new skill packages:

1. Activate the `create-skill` skill
2. Follow the instructions to prepare skill files
3. Use the `register_skill` tool to register the skill
4. The new skill can then be used via `activate`

<Note>
Built-in skills are stored in `templates/skills/` directory and automatically copied to `data/skills/` on startup.
</Note>
