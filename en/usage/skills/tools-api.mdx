---
title: "Skill Tools API"
---

The Skills system gives the Agent two tools. Both require the sandbox backend to be available ‚Äî they are not exposed to the Agent when Box is unavailable.

| Tool | Purpose |
|------|---------|
| `activate` | Activate an existing skill and pull its SKILL.md into context |
| `register_skill` | Register a directory prepared in the sandbox as a new skill |

---

## `activate`

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `skill_name` | string | Yes | Name of the skill to activate |

### Behavior

1. Looks up the name in the skill store
2. Returns SKILL.md content as a Tool Result and injects it into the current conversation context
3. Marks the skill as activated; it is reachable through `/workspace/.skills/{skill_name}/`

After activation:
- `read` / `glob` / `grep` can read every mounted skill
- `exec` / `write` / `edit` against a skill package is limited to **activated** skills

### Example Call

```json
{ "skill_name": "pdf" }
```

### Return Value

```json
{
  "activated": true,
  "skill_name": "pdf",
  "mount_path": "/workspace/.skills/pdf",
  "content": "<command-message>The \"pdf\" skill is activated</command-message>\n<skill-activation>\n  ...SKILL.md content...\n</skill-activation>"
}
```

### Errors

- `skill_name is required` ‚Äî name not provided
- `Skill "xxx" not found` ‚Äî skill does not exist

## `register_skill`

Registers a directory prepared under `/workspace` into the skill store.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Directory path in the sandbox; must be under `/workspace` |
| `name` | string | No | Override the skill name (defaults to `name` in SKILL.md or the directory name) |
| `display_name` | string | No | Override the display name |
| `description` | string | No | Override the description |
| `instructions` | string | No | Override the instruction body |

### Prerequisites

- Sandbox is enabled and available
- `path` is under `/workspace`
- The directory itself, or a subdirectory within two levels, contains `SKILL.md`

### Typical Flow

```
1. Agent prepares a directory in the sandbox:
   /workspace/my-skill/
   ‚îú‚îÄ‚îÄ SKILL.md
   ‚îî‚îÄ‚îÄ scripts/main.py

2. Agent calls register_skill(path="/workspace/my-skill")

3. The skill is written to the Box Runtime skill store,
   and any later session can activate it.
```

### Example Call

```json
{
  "path": "/workspace/my-skill",
  "name": "data-processor",
  "description": "Process and transform CSV/JSON data"
}
```

### Return Value

```json
{
  "registered": true,
  "skill_name": "data-processor",
  "source_path": "/workspace/my-skill",
  "skill": {
    "name": "data-processor",
    "display_name": "Data Processing Tool",
    "description": "Process and transform CSV/JSON data",
    "package_root": "/.../skills/data-processor"
  }
}
```

### Errors

- `path is required` ‚Äî path not provided
- `path must be under /workspace` ‚Äî path is not in the workspace
- `Directory does not exist` ‚Äî directory does not exist
- `No SKILL.md found in ...` ‚Äî no `SKILL.md` in the directory or within two levels of subdirectories
- `Multiple skill directories found...` ‚Äî the path contains multiple skill directories; point `path` to a specific subdirectory
