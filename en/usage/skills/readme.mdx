Skills is LangBot's dynamic skill system that allows AI Agents to activate and execute specific tasks on demand through Tool Call mechanism. Through Skills, you can enable Agents to automatically load domain-specific knowledge and tools when needed.

## What are Skills?

Skills is a lightweight extension mechanism that enables Agents to:

- Load domain-specific knowledge on demand via Tool Call
- Execute workflows for specific tasks
- Access predefined code and resources
- Dynamically combine multiple skills

Unlike plugins, Skills are more lightweight and flexible, suitable for defining workflows and instructions for specific tasks.

## Design Philosophy

### Industry Practices

The Skills system design draws from mainstream Agent extension patterns in the industry:

| System | Mechanism | Characteristics |
|--------|-----------|-----------------|
| **Claude Code** | Skills + Commands | Tool Call activation, Tool Result injection (protects KV Cache) |
| **OpenAI GPTs** | Actions + Knowledge | Separates tool calling from knowledge injection |
| **Cursor** | Rules + Tools | Independent configuration for rules and tools |

### Insights from Claude Code

Claude Code's Skills system has a key design decision: **skill content is injected through Tool Result, not by directly modifying the System Prompt**.

The reasons for this design:

1. **Protect KV Cache**: Directly modifying the System Prompt invalidates previous KV Cache, requiring recalculation every time
2. **On-demand Loading**: Skills are only injected when the Agent actively calls the `activate` tool, reducing unnecessary token consumption
3. **Incremental Context**: Tool Result is incremental content that doesn't affect existing cache

### LangBot's Design

LangBot fully adopts Claude Code's design pattern:

```
┌─────────────────────────────────────────┐
│         Agent (LLM)                      │
│                                          │
│  User request: "Help me analyze this     │
│  Python project"                         │
│                                          │
│  Decision: matches skill: python-...     │
│                                          │
│  Call: activate(skill_name="python-...") │
│                                          │
└───────────────────┬─────────────────────┘
                    │ Tool Call
                    ▼
┌─────────────────────────────────────────┐
│           activate Tool                  │
│                                          │
│  Returns: Tool Result                    │
│  - SKILL.md content                      │
│  - Mount path info                       │
│                                          │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│      Agent Continues (with skill context)│
│                                          │
│  - Use exec/read/write/edit to operate   │
│    skill package                         │
│  - Execute tasks following instructions  │
│  - Can call activate multiple times      │
│                                          │
└─────────────────────────────────────────┘
```

### Why This Design?

**1. KV Cache Protection**

Skill instructions are returned through Tool Result as incremental content, not invalidating existing KV Cache.

**2. Progressive Disclosure**

- `activate` tool description only shows skill metadata (name, description)
- Full instructions are obtained after activation via Tool Result
- Agent can choose whether to activate based on needs

**3. Unified Interaction Pattern**

- Consistent semantics with other tool calls
- More aligned with Agent design patterns
- Friendlier for models supporting Function Calling

**4. Requires Sandbox Backend**

`activate` and `register_skill` tools require sandbox backend availability:
- Skill package is mounted to `/workspace/.skills/{skill-name}/`
- Agent uses `exec`/`read`/`write`/`edit`/`glob`/`grep` to operate skill files

## Skill File Structure

Each Skill is a directory containing a `SKILL.md` file:

```
data/skills/
├── my-skill/
│   ├── SKILL.md           # Required: skill definition file
│   ├── scripts/           # Optional: script files
│   │   └── analyze.py
│   └── templates/         # Optional: template files
│       └── report.md
└── another-skill/
    └── SKILL.md
```

### SKILL.md Format

Use YAML frontmatter to define metadata:

```markdown
---
name: python-analyzer
display_name: Python Project Analyzer
description: Analyze Python project structure, code quality and dependencies
---

## Use Cases

When users need to analyze Python projects, follow these steps:

1. Use the `read` tool to read project structure
2. Use `exec` to run analysis scripts
3. Generate analysis report

## Available Scripts

- `scripts/analyze.py`: Main analysis script
- `scripts/dependency_check.py`: Dependency checking

## Output Format

Analysis report should include:
- Project structure overview
- Code quality score
- Dependency graph
```

## Activation Mechanism

### Tool Call Activation

Agent activates a skill by calling the `activate` tool:

```json
{
  "skill_name": "python-analyzer"
}
```

### Effects After Activation

1. **Tool Result Injection**: SKILL.md content is returned as Tool Result, injected into context
2. **Path Mounting**: Skill package is mounted to `/workspace/.skills/{skill-name}/`
3. **Tool Access**: Agent can use `exec`/`read`/`write`/`edit`/`glob`/`grep` to operate skill files

### Return Content Example

```
<command-message>The "python-analyzer" skill is activated</command-message>
<skill-activation>
<skill-name>python-analyzer</skill-name>
<mount-path>/workspace/.skills/python-analyzer</mount-path>
<package-root>/path/to/skill</package-root>

## Instructions
...SKILL.md content...

## Runtime Context
The skill package is mounted at /workspace/.skills/python-analyzer. Use the standard tools to interact with it:
- Use `read` to inspect files under /workspace/.skills/python-analyzer
- Use `exec` with workdir set to /workspace/.skills/python-analyzer to run commands
- Use `write` and `edit` on that path when the instructions require updating files
</skill-activation>
```

## Built-in Tools

The Skills system provides two tools for Agents:

| Tool | Function | Needs Sandbox |
|------|----------|---------------|
| `activate` | Activate a skill | Yes |
| `register_skill` | Register skill from sandbox directory | Yes |

<Note>
These tools require a sandbox backend (Docker/Nsjail/E2B) to be available. If no backend is available, the tools will not be provided to the Agent.
</Note>

## Skill Types

### Prompt-Only Skills

Contains only instructions, no additional files:

```markdown
---
name: code-reviewer
display_name: Code Review Assistant
description: Review code and provide improvement suggestions
---

## Review Steps

1. Analyze code structure and style
2. Check for potential bugs and security issues
3. Provide improvement suggestions

## Review Criteria

- Code readability
- Performance optimization
- Security best practices
```

### File Package Skills

Contains scripts, templates, and other resource files:

```
data-analyzer/
├── SKILL.md
├── scripts/
│   ├── analyze.py
│   └── visualize.py
├── templates/
│   └── report.md
└── requirements.txt
```

## Quick Start

### 1. Create Your First Skill

Create a skill in the `data/skills/` directory:

```bash
mkdir -p data/skills/my-first-skill
```

Create `SKILL.md`:

```markdown
---
name: my-first-skill
display_name: My First Skill
description: A simple example skill
---

## Task Description

This is an example skill demonstrating basic skill functionality.

## Execution Steps

1. Understand user request
2. Execute corresponding operations
3. Return results
```

### 2. Test Activation

Ensure sandbox backend is available, then send a request matching the skill description to the Agent. The Agent will automatically call the `activate` tool to activate the skill.

## Creating New Skills

### Method 1: Use Built-in create-skill Skill

LangBot includes a built-in `create-skill` skill for guiding the creation of new skill packages:

1. Call `activate(skill_name="create-skill")`
2. Follow the returned instructions to prepare skill files
3. Use the `register_skill` tool to register the skill

### Method 2: Manual Creation

1. Create skill directory under sandbox `/workspace`
2. Write `SKILL.md` and related files
3. Call `register_skill(path="/workspace/my-skill")` to register

## Built-in Skills

LangBot automatically copies built-in skills from `templates/skills/` to `data/skills/` on startup:

| Skill | Description |
|-------|-------------|
| `create-skill` | Skill for guiding creation of new skills |

## Best Practices

### 1. Skill Naming

- Use lowercase letters, numbers, hyphens, underscores
- Names should be concise and descriptive
- Examples: `python-analyzer`, `data-visualizer`

### 2. Description Writing

- Clearly state the skill's purpose and applicable scenarios
- Help LLM correctly determine when to activate
- Description is shown in the `activate` tool's `<available_skills>` list

### 3. Instruction Design

- Provide clear execution steps
- Explain available resources and tools
- Define output formats

### 4. Resource Organization

- Put scripts in `scripts/` directory
- Put templates in `templates/` directory
- Keep structure clear
